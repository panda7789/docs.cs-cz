# <a name="cannot-implement-interfacename1membername-because-its-implementation-could-conflict-with-the-implementation-for-interfacename2membername-for-some-type-arguments"></a><span data-ttu-id="6c388-102">Nemůže implementovat '\<interfacename1 >.\< jméno >' protože jeho implementace by mohla konfliktu s implementací pro '\<interfacename2 >.\< jméno >' u některých argumentů typů</span><span class="sxs-lookup"><span data-stu-id="6c388-102">Cannot implement '\<interfacename1>.\<membername>' because its implementation could conflict with the implementation for '\<interfacename2>.\<membername>' for some type arguments</span></span>
<span data-ttu-id="6c388-103">Třída implementuje více než jedno obecné rozhraní, z nichž jeden z nich dědí z jiné a dvě implementace člena rozhraní mohou být v konfliktu pro určité hodnoty argumentů typu.</span><span class="sxs-lookup"><span data-stu-id="6c388-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="6c388-104">Následující příkazy mohou vygenerovat tuto chybu.</span><span class="sxs-lookup"><span data-stu-id="6c388-104">The following statements can generate this error.</span></span>  
  
```vb  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="6c388-105">Vzhledem k tomu, že `iFace2` dědí z `iFace1` pomocí vlastního parametru typu (`u`), `testClass` by implementoval dvě verze `iFace1.testSub` se stejnými podpisy, pokud byl stejný argument typu předán do `y` a `z`.</span><span class="sxs-lookup"><span data-stu-id="6c388-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="6c388-106">Tím by vznikla nejednoznačnost, o které verzi přistupuje.</span><span class="sxs-lookup"><span data-stu-id="6c388-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="6c388-107">**ID chyby:** BC32125</span><span class="sxs-lookup"><span data-stu-id="6c388-107">**Error ID:** BC32125</span></span>

- <span data-ttu-id="6c388-109">Změňte strukturu dědičnosti rozhraní tak, aby `iFace1` nebylo možné implementovat se dvěma různými argumenty typu.</span><span class="sxs-lookup"><span data-stu-id="6c388-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>
