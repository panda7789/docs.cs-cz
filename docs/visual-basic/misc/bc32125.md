# <a name="cannot-implement-interfacename1membername-because-its-implementation-could-conflict-with-the-implementation-for-interfacename2membername-for-some-type-arguments"></a><span data-ttu-id="e498f-102">Nelze implementovat ' \<interfacename1> . \<membername> ', protože jeho implementace by mohla být v konfliktu s implementací pro ' \<interfacename2> . \<membername> ' pro některé argumenty typu.</span><span class="sxs-lookup"><span data-stu-id="e498f-102">Cannot implement '\<interfacename1>.\<membername>' because its implementation could conflict with the implementation for '\<interfacename2>.\<membername>' for some type arguments</span></span>
<span data-ttu-id="e498f-103">Třída implementuje více než jedno obecné rozhraní, z nichž jeden z nich dědí z jiné a dvě implementace člena rozhraní mohou být v konfliktu pro určité hodnoty argumentů typu.</span><span class="sxs-lookup"><span data-stu-id="e498f-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="e498f-104">Následující příkazy mohou vygenerovat tuto chybu.</span><span class="sxs-lookup"><span data-stu-id="e498f-104">The following statements can generate this error.</span></span>  
  
```vb  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="e498f-105">Protože `iFace2` dědí z `iFace1` použití vlastního parametru typu ( `u` ), `testClass` by implementoval dvě verze `iFace1.testSub` s identickými podpisy, pokud byl stejný argument typu předán do `y` a `z` .</span><span class="sxs-lookup"><span data-stu-id="e498f-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="e498f-106">Tím by vznikla nejednoznačnost, o které verzi přistupuje.</span><span class="sxs-lookup"><span data-stu-id="e498f-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="e498f-107">**ID chyby:** BC32125</span><span class="sxs-lookup"><span data-stu-id="e498f-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="e498f-108">Oprava této chyby</span><span class="sxs-lookup"><span data-stu-id="e498f-108">To correct this error</span></span>  
  
- <span data-ttu-id="e498f-109">Změňte strukturu dědičnosti rozhraní tak, aby nebyla `iFace1` implementována se dvěma různými argumenty typu.</span><span class="sxs-lookup"><span data-stu-id="e498f-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="e498f-110">-nebo-</span><span class="sxs-lookup"><span data-stu-id="e498f-110">-or-</span></span>
