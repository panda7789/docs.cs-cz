- <span data-ttu-id="e3d21-112">Oba mají členy, které mohou zahrnovat konstruktory, metody, vlastnosti, pole, konstanty, výčty, události a obslužné rutiny událostí.</span><span class="sxs-lookup"><span data-stu-id="e3d21-112">Both have members, which can include constructors, methods, properties, fields, constants, enumerations, events, and event handlers.</span></span> <span data-ttu-id="e3d21-113">Nepleťte si však tyto členy s deklarovanými *prvky* struktury.</span><span class="sxs-lookup"><span data-stu-id="e3d21-113">However, do not confuse these members with the declared *elements* of a structure.</span></span>  
  
- <span data-ttu-id="e3d21-114">Členové obou můžou mít individuální úrovně přístupu.</span><span class="sxs-lookup"><span data-stu-id="e3d21-114">Members of both can have individualized access levels.</span></span> <span data-ttu-id="e3d21-115">Například jeden člen může být deklarován `Public` a druhý `Private` .</span><span class="sxs-lookup"><span data-stu-id="e3d21-115">For example, one member can be declared `Public` and another `Private`.</span></span>  
  
- <span data-ttu-id="e3d21-116">Oba můžou implementovat rozhraní.</span><span class="sxs-lookup"><span data-stu-id="e3d21-116">Both can implement interfaces.</span></span>

## <a name="differences"></a><span data-ttu-id="e3d21-120">Rozdíly</span><span class="sxs-lookup"><span data-stu-id="e3d21-120">Differences</span></span>  
 <span data-ttu-id="e3d21-121">Struktury a třídy se liší v následujících údajích:</span><span class="sxs-lookup"><span data-stu-id="e3d21-121">Structures and classes differ in the following particulars:</span></span>  
  
- <span data-ttu-id="e3d21-122">Struktury jsou *typy hodnot*; třídy jsou *odkazové typy*.</span><span class="sxs-lookup"><span data-stu-id="e3d21-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="e3d21-123">Proměnná typu struktury obsahuje data struktury místo toho, aby obsahovala odkaz na data jako typ třídy.</span><span class="sxs-lookup"><span data-stu-id="e3d21-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
- <span data-ttu-id="e3d21-124">Struktury používají alokaci zásobníku; třídy používají přidělení haldy.</span><span class="sxs-lookup"><span data-stu-id="e3d21-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
- <span data-ttu-id="e3d21-125">Všechny prvky struktury jsou `Public` ve výchozím nastavení. proměnné třídy a konstanty jsou `Private` ve výchozím nastavení, zatímco jiné členy třídy jsou `Public` ve výchozím nastavení.</span><span class="sxs-lookup"><span data-stu-id="e3d21-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="e3d21-126">Toto chování pro členy třídy zajišťuje kompatibilitu s výchozím systémem Visual Basic 6,0.</span><span class="sxs-lookup"><span data-stu-id="e3d21-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
- <span data-ttu-id="e3d21-127">Struktura musí mít alespoň jednu nesdílenou proměnnou nebo nevlastní nesdílenou, nevlastní prvek události; Třída může být zcela prázdná.</span><span class="sxs-lookup"><span data-stu-id="e3d21-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
- <span data-ttu-id="e3d21-128">Prvky struktury nelze deklarovat jako `Protected` ; Členové třídy mohou.</span><span class="sxs-lookup"><span data-stu-id="e3d21-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
- <span data-ttu-id="e3d21-129">Procedura struktury může zpracovávat události pouze v případě, že se jedná o [sdílenou](../../../language-reference/modifiers/shared.md) `Sub` proceduru, a to pouze pomocí [příkazu AddHandler](../../../language-reference/statements/addhandler-statement.md); jakákoli procedura třídy může zpracovávat události pomocí klíčového slova [Handles](../../../language-reference/statements/handles-clause.md) nebo `AddHandler` příkazu.</span><span class="sxs-lookup"><span data-stu-id="e3d21-129">A structure procedure can handle events only if it is a [Shared](../../../language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="e3d21-130">Další informace najdete v tématu [události](../events/index.md).</span><span class="sxs-lookup"><span data-stu-id="e3d21-130">For more information, see [Events](../events/index.md).</span></span>  
  
- <span data-ttu-id="e3d21-131">Deklarace proměnných struktury nemůžou určovat Inicializátory ani počáteční velikosti polí. deklarace proměnných třídy mohou.</span><span class="sxs-lookup"><span data-stu-id="e3d21-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>  
  
- <span data-ttu-id="e3d21-132">Struktury implicitně dědí z <xref:System.ValueType?displayProperty=nameWithType> třídy a nemůžou dědit z žádného jiného typu; třídy mohou dědit z jakékoliv třídy nebo třídy kromě <xref:System.ValueType?displayProperty=nameWithType> .</span><span class="sxs-lookup"><span data-stu-id="e3d21-132">Structures implicitly inherit from the <xref:System.ValueType?displayProperty=nameWithType> class and cannot inherit from any other type; classes can inherit from any class or classes other than <xref:System.ValueType?displayProperty=nameWithType>.</span></span>  
  
- <span data-ttu-id="e3d21-133">Struktury nejsou dědičné; třídy jsou.</span><span class="sxs-lookup"><span data-stu-id="e3d21-133">Structures are not inheritable; classes are.</span></span>  
  
- <span data-ttu-id="e3d21-134">Struktury nejsou nikdy ukončeny, takže modul CLR (Common Language Runtime) nikdy nevolá <xref:System.Object.Finalize%2A> metodu v žádné struktuře. třídy jsou ukončeny systémem uvolňování paměti (GC), který volá <xref:System.Object.Finalize%2A> třídu, když zjistí, že neexistují žádné aktivní odkazy.</span><span class="sxs-lookup"><span data-stu-id="e3d21-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
- <span data-ttu-id="e3d21-135">Struktura nevyžaduje konstruktor. Třída dělá.</span><span class="sxs-lookup"><span data-stu-id="e3d21-135">A structure does not require a constructor; a class does.</span></span>  
  
- <span data-ttu-id="e3d21-136">Struktury mohou mít nesdílené konstruktory pouze v případě, že převezmou parametry; třídy mohou mít s parametry nebo bez nich.</span><span class="sxs-lookup"><span data-stu-id="e3d21-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="e3d21-137">Každá struktura má implicitní veřejný konstruktor bez parametrů.</span><span class="sxs-lookup"><span data-stu-id="e3d21-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="e3d21-138">Tento konstruktor inicializuje všechny prvky dat struktury na jejich výchozí hodnoty.</span><span class="sxs-lookup"><span data-stu-id="e3d21-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="e3d21-139">Toto chování nelze předefinovat.</span><span class="sxs-lookup"><span data-stu-id="e3d21-139">You cannot redefine this behavior.</span></span>  
  
## <a name="instances-and-variables"></a><span data-ttu-id="e3d21-140">Instance a proměnné</span><span class="sxs-lookup"><span data-stu-id="e3d21-140">Instances and Variables</span></span>  
 <span data-ttu-id="e3d21-141">Vzhledem k tomu, že struktury jsou typy hodnot, každá proměnná struktury je trvale svázána s instancí jednotlivé struktury.</span><span class="sxs-lookup"><span data-stu-id="e3d21-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="e3d21-142">Ale třídy jsou odkazové typy a proměnná objektu může odkazovat na různé instance třídy v různých časech.</span><span class="sxs-lookup"><span data-stu-id="e3d21-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="e3d21-143">Toto rozlišení má vliv na využití struktur a tříd následujícími způsoby:</span><span class="sxs-lookup"><span data-stu-id="e3d21-143">This distinction affects your usage of structures and classes in the following ways:</span></span>

     <span data-ttu-id="e3d21-157">Prvky struktury jsou však izolované v rámci své vlastní instance.</span><span class="sxs-lookup"><span data-stu-id="e3d21-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="e3d21-158">Změny jejich hodnot se neprojeví v žádné jiné proměnné struktury, ani v jiných instancích stejné `Structure` deklarace.</span><span class="sxs-lookup"><span data-stu-id="e3d21-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>
