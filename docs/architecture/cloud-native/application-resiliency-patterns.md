## <a name="circuit-breaker-pattern"></a><span data-ttu-id="96c5e-146">Vzorek pro přerušení okruhu</span><span class="sxs-lookup"><span data-stu-id="96c5e-146">Circuit breaker pattern</span></span>

<span data-ttu-id="96c5e-147">I když vzor opakování může přispět k vyřazení žádosti entangled v částečném selhání, existují situace, kdy selhání může být způsobeno neočekávanými událostmi, které budou vyžadovat přeložení delší dobu.</span><span class="sxs-lookup"><span data-stu-id="96c5e-147">While the retry pattern can help salvage a request entangled in a partial failure, there are situations where failures can be caused by unanticipated events that will require longer periods of time to resolve.</span></span> <span data-ttu-id="96c5e-148">Tyto chyby můžou být různě závažné, od částečného výpadku připojení až po úplné selhání služby.</span><span class="sxs-lookup"><span data-stu-id="96c5e-148">These faults can range in severity from a partial loss of connectivity to the complete failure of a service.</span></span> <span data-ttu-id="96c5e-149">V těchto situacích je bezúčelné pro aplikaci, aby se nepřetržitě opakovala operace, která není pravděpodobně úspěšná.</span><span class="sxs-lookup"><span data-stu-id="96c5e-149">In these situations, it's pointless for an application to continually retry an operation that is unlikely to succeed.</span></span>

<span data-ttu-id="96c5e-150">Kvůli horšímu provádění operací nepřetržitého opakování při opakovaném pokusu o nereagující službu vás může přejít na scénář útoku na nereagující službu, kde jste službu zaplavi pomocí nepřetržitých volání vyčerpání prostředků, jako jsou paměť, vlákna a databázová připojení, což způsobuje selhání nesouvisejících částí systému, které používají stejné prostředky.</span><span class="sxs-lookup"><span data-stu-id="96c5e-150">To make things worse, executing continual retry operations on a non-responsive service can move you into a self-imposed denial of service scenario where you flood your service with continual calls exhausting resources such as memory, threads and database connections, causing failure in unrelated parts of the system that use the same resources.</span></span>

<span data-ttu-id="96c5e-151">V těchto situacích by bylo vhodnější, že operace selže okamžitě a pokusí se o vyvolání služby pouze v případě, že je pravděpodobně úspěšná.</span><span class="sxs-lookup"><span data-stu-id="96c5e-151">In these situations, it would be preferable for the operation to fail immediately and only attempt to invoke the service if it's likely to succeed.</span></span>
