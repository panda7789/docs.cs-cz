# <a name="microservices-architecture"></a><span data-ttu-id="999a3-103">Architektura mikroslužeb</span><span class="sxs-lookup"><span data-stu-id="999a3-103">Microservices architecture</span></span>

<span data-ttu-id="999a3-104">Jak název napovídá, architektura mikroslužeb je přístup k vytváření serverové aplikace jako sadu malých služeb.</span><span class="sxs-lookup"><span znamená, že architektura mikroslužeb je hlavně orientována na back-end, i když přístup se používá také pro front-end.</span><span class="sxs-lookup"><span data-stu-id="999a3-105">That means a microservices architecture is mainly oriented to the back-end, although the approach is also being used for the front end.</span></span> <span data-ttu-id="999a3-106">Každá služba běží ve vlastním procesu a komunikuje s jinými procesy pomocí protokolů, jako je HTTP/HTTPS, WebSockets nebo [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span><span class="sxs-lookup"><span data-stu-id="999a3-106">Each service runs in its own process and communicates with other processes using protocols such as HTTP/HTTPS, WebSockets, or [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span></span> <span data-ttu-id="999a3-107">Každá mikroslužba implementuje konkrétní doménu end nebo obchodní schopnosti v by každá mikroslužba měla vlastnit svůj související datový model domény a logiku domény (suverenita a decentralizovaná správa dat) a mohla by být založena na různých technologiích pro ukládání dat (SQL, NoSQL) a různých programovacích jazycích.</span><span class="sxs-lookup"><span data-stu-id="999a3-108">Finally, each microservice should own its related domain data model and domain logic (sovereignty and decentralized data management) and could be based on different data storage technologies (SQL, NoSQL) and different programming languages.</span></span>

<span data-ttu-id="999a3-109">Jakou velikost by měla být mikroslužba?</span><span class="sxs-lookup"><span data-stu-id="999a3-109">What size should a microservice be?</span></span> <span data-ttu-id="999a3-110">Při vývoji mikroslužby velikost by neměla být důležitým bodem.</span><span class="sxs-lookup"><span data-stu-id="999a3-110">When developing a microservice, size shouldn't be the important point.</span></span> <span data-ttu-id="999a3-111">Místo toho by mělo být důležité vytvořit volně vázané služby, abyste měli autonomii vývoje, nasazení a škálování pro každou službu.</span><span class="sxs-lookup"><span data-stu-id="999a3-111">Instead, the important point should be to create loosely coupled services so you have autonomy of development, deployment, and scale, for each service.</span></span> <span data-ttu-id="999a3-112">Samozřejmě při identifikaci a navrhování mikroslužeb, měli byste se pokusit, aby byly co nejmenší, pokud nemáte příliš mnoho přímých závislostí s jinými mikroslužbami.</span><span class="sxs-lookup"><span data-stu-id="999a3-112">Of course, when identifying and designing microservices, you should try to make them as small as possible as long as you don't have too many direct dependencies with other microservices.</span></span> <span data-ttu-id="999a3-113">Důležitější než velikost mikroslužby je vnitřní soudržnost, kterou musí mít, a její nezávislost na jiných službách.</span><span class="sxs-lookup"><span data-stu-id="999a3-113">More important than the size of the microservice is the internal cohesion it must have and its independence from other services.</span></span>

<span data-ttu-id="999a3-114">Proč architektura mikroslužeb?</span><span class="sxs-lookup"><span data-stu-id="999a3-114">Why a microservices architecture?</span></span> <span data-ttu-id="999a3-115">Stručně řečeno, poskytuje dlouhodobou agilitu.</span><span class="sxs-lookup"><span data-stu-id="999a3-115">In short, it provides long-term agility.</span></span> <span data-ttu-id="999a3-116">Mikroslužby umožňují lepší udržovatelnost ve složitých, velkých a vysoce škálovatelných systémech tím, že umožňují vytvářet aplikace založené na mnoha nezávisle nasaditelných službách, které mají podrobné a autonomní životní cykly.</span><span class="sxs-lookup"><span data-stu-id="999a3-116">Microservices enable better maintainability in complex, large, and highly-scalable systems by letting you create applications based on many independently deployable services that each have granular and autonomous lifecycles.</span></span>

<span data-ttu-id="999a3-117">Jako další výhodu mikroslužeb můžete škálovat nezávisle.</span><span class="sxs-lookup"><span data-stu-id="999a3-117">As an additional benefit, microservices can scale out independently.</span></span> <span data-ttu-id="999a3-118">Místo toho, aby s jednu monolitickou aplikaci, kterou je nutné horizontální navýšení kapacity jako celek, můžete místo toho horizontální navýšení kapacity konkrétní mikroslužeb.</span><span class="sxs-lookup"><span data-stu-id="999a3-118">Instead of having a single monolithic application that you must scale out as a unit, you can instead scale out specific microservices.</span></span> <span data-ttu-id="999a3-119">Tímto způsobem můžete škálovat pouze funkční oblast, která potřebuje větší výpočetní výkon nebo šířku pásma sítě pro podporu poptávky, spíše než horizontální navýšení kapacity jiných oblastí aplikace, které není nutné škálovat.</span><span class="sxs-lookup"><span data-stu-id="999a3-119">That way, you can scale just the functional area that needs more processing power or network bandwidth to support demand, rather than scaling out other areas of the application that don't need to be scaled.</span></span> <span data-ttu-id="999a3-120">To znamená úsporu nákladů, protože potřebujete méně hardwaru.</span><span class="sxs-lookup"><span data-stu-id="999a3-120">That means cost savings because you need less hardware.</span></span>

<span data-ttu-id="999a3-127">Navrhování jemně odstupňovaných aplikací založených na mikroslužbách umožňuje průběžnou integraci a průběžné postupy doručování.</span><span class="sxs-lookup"><span data-stu-id="999a3-127">Architecting fine-grained microservices-based applications enables continuous integration and continuous delivery practices.</span></span> <span data-ttu-id="999a3-128">Také urychluje dodávku nových funkcí do aplikace.</span><span class="sxs-lookup"><span data-stu-id="999a3-128">It also accelerates delivery of new functions into the application.</span></span> <span data-ttu-id="999a3-129">Jemně odstupňované složení aplikací také umožňuje spustit a otestovat mikroslužeb v izolaci a vyvíjet je autonomně při zachování jasné smlouvy mezi nimi.</span><span class="sxs-lookup"><span data-stu-id="999a3-129">Fine-grained composition of applications also allows you to run and test microservices in isolation, and to evolve them autonomously while maintaining clear contracts between them.</span></span> <span data-ttu-id="999a3-130">Pokud nezměníte rozhraní nebo smlouvy, můžete změnit interní implementaci jakékoli mikroslužby nebo přidat nové funkce bez přerušení jiných mikroslužeb.</span><span class="sxs-lookup"><span data-stu-id="999a3-130">As long as you don't change the interfaces or contracts, you can change the internal implementation of any microservice or add new functionality without breaking other microservices.</span></span>

<span data-ttu-id="999a3-131">Následující jsou důležité aspekty, které umožňují úspěch při vstupu do výroby pomocí systému založeného na mikroslužbách:</span><span class="sxs-lookup"><span data-stu-id="999a3-131">The following are important aspects to enable success in going into production with a microservices-based system:</span></span>

- <span data-ttu-id="999a3-132">Monitorování a zdravotní kontroly služeb a infrastruktury.</span><span class="sxs-lookup"><span data-stu-id="999a3-132">Monitoring and health checks of the services and infrastructure.</span></span>

- <span data-ttu-id="999a3-133">Škálovatelná infrastruktura pro služby (to znamená cloud a orchestrátory).</span><span class="sxs-lookup"><span data-stu-id="999a3-133">Scalable infrastructure for the services (that is, cloud and orchestrators).</span></span>

- <span data-ttu-id="999a3-134">Návrh zabezpečení a implementace na více úrovních: ověřování, autorizace, správa tajných kódů, zabezpečená komunikace atd.</span><span class="sxs-lookup"><span data-stu-id="999a3-134">Security design and implementation at multiple levels: authentication, authorization, secrets management, secure communication, etc.</span></span>

- <span data-ttu-id="999a3-135">Rychlé dodání aplikace, obvykle s různými týmy se zaměřením na různé mikroslužby.</span><span class="sxs-lookup"><span data-stu-id="999a3-135">Rapid application delivery, usually with different teams focusing on different microservices.</span></span>

- <span data-ttu-id="999a3-136">DevOps a CI/CD postupy a infrastruktury.</span><span class="sxs-lookup"><span data-stu-id="999a3-136">DevOps and CI/CD practices and infrastructure.</span></span>
