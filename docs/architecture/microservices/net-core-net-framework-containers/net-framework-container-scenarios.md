<span data-ttu-id="bfa05-106">Kontejnery Docker můžete chtít použít jenom ke zjednodušení nasazení, i když nevytváříte mikroslužby.</span><span class="sxs-lookup"><span data-stu-id="bfa05-106">You might want to use Docker containers just to simplify deployment, even if you are not creating microservices.</span></span> <span data-ttu-id="bfa05-107">Například můžete chtít zlepšit pracovní postup DevOps pomocí Docker – kontejnery vám poskytnou lepší izolovaná testovací prostředí a můžou také eliminovat problémy s nasazením způsobené chybějícími závislostmi při přesunu do produkčního prostředí.</span><span class="sxs-lookup"><span data-stu-id="bfa05-107">For example, perhaps you want to improve your DevOps workflow with Docker—containers can give you better isolated test environments and can also eliminate deployment issues caused by missing dependencies when you move to a production environment.</span></span> <span data-ttu-id="bfa05-108">V takových případech, i když nasazujete aplikaci monolitické, má smysl použít pro vaše aktuální aplikace .NET Framework kontejnery Docker a Windows.</span><span class="sxs-lookup"><span data-stu-id="bfa05-108">In cases like these, even if you are deploying a monolithic application, it makes sense to use Docker and Windows Containers for your current .NET Framework applications.</span></span>

<span data-ttu-id="bfa05-115">[Sada Compatibility Pack systému Windows](../../../core/porting/windows-compat-pack.md) navíc rozšiřuje plochu rozhraní API, která je k dispozici pro .NET Standard 2,0 ve Windows.</span><span class="sxs-lookup"><span data-stu-id="bfa05-115">In addition, the [Windows Compatibility Pack](../../../core/porting/windows-compat-pack.md) extends the API surface available for .NET Standard 2.0 on Windows.</span></span> <span data-ttu-id="bfa05-116">Tento balíček umožňuje opětovně kompilovat stávající kód, aby .NET Standard 2. x s minimální nebo žádnou úpravou pro spuštění ve Windows.</span><span class="sxs-lookup"><span data-stu-id="bfa05-116">This pack allows recompiling most existing code to .NET Standard 2.x with little or no modification, to run on Windows.</span></span>

<span data-ttu-id="bfa05-117">Nicméně i s výjimečným průběhem od .NET Standard 2,0 a .NET Core 2,1 mohou nastat případy, kdy některé balíčky NuGet potřebují ke spuštění Windows a nemusí podporovat .NET Core.</span><span class="sxs-lookup"><span data-stu-id="bfa05-117">However, even with that exceptional progression since .NET Standard 2.0 and .NET Core 2.1, there might be cases where certain NuGet packages need Windows to run and might not support .NET Core.</span></span> <span data-ttu-id="bfa05-118">Pokud jsou tyto balíčky pro vaši aplikaci důležité, budete je muset použít .NET Framework v kontejnerech Windows.</span><span class="sxs-lookup"><span data-stu-id="bfa05-118">If those packages are critical for your application, then you will need to use .NET Framework on Windows Containers.</span></span>
