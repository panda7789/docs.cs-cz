<span data-ttu-id="c9c07-160">Vynikající [návrh založený na doméně](https://domainlanguage.com/ddd/) Eric Evans uvádí následující informace o vrstvě doménového modelu a vrstvě aplikace.</span><span class="sxs-lookup"><span data-stu-id="c9c07-160">Eric Evans's excellent book [Domain Driven Design](https://domainlanguage.com/ddd/) says the following about the domain model layer and the application layer.</span></span>

<span data-ttu-id="c9c07-173">I v případě, že je důležité dodržovat princip ignorování trvalosti pro doménový model, neměli byste ignorovat obavy o trvalosti.</span><span class="sxs-lookup"><span data-stu-id="c9c07-173">Even when it is important to follow the Persistence Ignorance principle for your Domain model, you should not ignore persistence concerns.</span></span> <span data-ttu-id="c9c07-174">Pořád je důležité pochopit fyzický datový model a jak se mapuje na model objektu entity.</span><span class="sxs-lookup"><span data-stu-id="c9c07-174">It is still very important to understand the physical data model and how it maps to your entity object model.</span></span> <span data-ttu-id="c9c07-175">V opačném případě můžete vytvořit neproveditelné návrhy.</span><span class="sxs-lookup"><span data-stu-id="c9c07-175">Otherwise you can create impossible designs.</span></span>

<span data-ttu-id="c9c07-176">To neznamená, že můžete vytvořit model navržený pro relační databázi a přímo ho přesunout do NoSQL nebo databáze orientované na dokument.</span><span class="sxs-lookup"><span data-stu-id="c9c07-176">Also, this does not mean you can take a model designed for a relational database and directly move it to a NoSQL or document-oriented database.</span></span> <span data-ttu-id="c9c07-177">V některých modelech entit se model může přizpůsobit, ale obvykle to není.</span><span class="sxs-lookup"><span data-stu-id="c9c07-177">In some entity models, the model might fit, but usually it does not.</span></span> <span data-ttu-id="c9c07-178">K dispozici jsou i omezení, která musí splňovat váš model entity, a to na základě technologie úložiště i technologie ORM.</span><span class="sxs-lookup"><span data-stu-id="c9c07-178">There are still constraints that your entity model must adhere to, based both on the storage technology and ORM technology.</span></span>

### <a name="the-application-layer"></a><span data-ttu-id="c9c07-179">Aplikační vrstva</span><span class="sxs-lookup"><span data-stu-id="c9c07-179">The application layer</span></span>

<span data-ttu-id="c9c07-180">Když se přesunete na aplikační vrstvu, můžeme znovu citovat [Návrh řízený](https://domainlanguage.com/ddd/)z Eric Evans v knize založené na doméně:</span><span class="sxs-lookup"><span data-stu-id="c9c07-180">Moving on to the application layer, we can again cite Eric Evans's book [Domain Driven Design](https://domainlanguage.com/ddd/):</span></span>
