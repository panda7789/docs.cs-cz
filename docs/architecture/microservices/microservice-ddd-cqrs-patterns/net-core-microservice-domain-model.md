## <a name="encapsulate-data-in-the-domain-entities"></a><span data-ttu-id="12501-147">Zapouzdřit data v doménových entitách</span><span class="sxs-lookup"><span data-stu-id="12501-147">Encapsulate data in the Domain Entities</span></span>

<span data-ttu-id="12501-148">Běžným problémem v modelech entit je, že zveřejňují vlastnosti navigace kolekce jako veřejně přístupné typy seznamů.</span><span class="sxs-lookup"><span data-stu-id="12501-148">A common problem in entity models is that they expose collection navigation properties as publicly accessible list types.</span></span> <span data-ttu-id="12501-149">To umožňuje vývojářům spolupracovníkům manipulovat s obsahem těchto typů kolekcí, což může obejít důležité obchodní pravidla související s kolekcí, což může opustit objekt v neplatném stavu.</span><span class="sxs-lookup"><span data-stu-id="12501-149">This allows any collaborator developer to manipulate the contents of these collection types, which may bypass important business rules related to the collection, possibly leaving the object in an invalid state.</span></span> <span data-ttu-id="12501-150">Řešením je vystavit přístup jen pro čtení k souvisejícím kolekcím a explicitně poskytnout metody, které definují způsoby, kterými je můžou klienti manipulovat.</span><span class="sxs-lookup"><span data-stu-id="12501-150">The solution to this is to expose read-only access to related collections and explicitly provide methods that define ways in which clients can manipulate them.</span></span>

<span data-ttu-id="12501-151">V předchozím kódu si všimněte, že mnoho atributů je pouze pro čtení nebo je soukromé a lze je aktualizovat pouze metodami třídy, takže jakákoli aktualizace bere v úvahu invariantní v podnikové doméně a logiku zadanou v rámci metod třídy.</span><span class="sxs-lookup"><span data-stu-id="12501-151">In the previous code, note that many attributes are read-only or private and are only updatable by the class methods, so any update considers business domain invariants and logic specified within the class methods.</span></span>

<span data-ttu-id="12501-152">Například následující vzory DDD **byste neměli provádět**  z jakékoli metody obslužné rutiny příkazu nebo třídy vrstvy aplikace (ve skutečnosti by to nemělo být možné):</span><span class="sxs-lookup"><span data-stu-id="12501-152">For example, following DDD patterns, **you should *not* do the following** from any command handler method or application layer class (actually, it should be impossible for you to do so):</span></span>

```csharp

//... (WRONG) Some code with business logic out of the domain classes ...

//... (WRONG) Accessing the OrderItems collection directly from the application layer // or command handlers

```

<span data-ttu-id="12501-153">V tomto případě je metoda Add čistě operací pro přidání dat s přímým přístupem ke kolekci OrderItems.</span><span class="sxs-lookup"><span data-stu-id="12501-153">In this case, the Add method is purely an operation to add data, with direct access to the OrderItems collection.</span></span> <span data-ttu-id="12501-154">Proto se většina doménové logiky, pravidel nebo ověření souvisejících s touto operací s podřízenými entitami rozprostře napříč aplikační vrstvou (obslužné rutiny příkazů a řadiče webového rozhraní API).</span><span class="sxs-lookup"><span data-stu-id="12501-154">Therefore, most of the domain logic, rules, or validations related to that operation with the child entities will be spread across the application layer (command handlers and Web API controllers).</span></span>

<span data-ttu-id="12501-155">Pokud překročíte agregovanou kořenovou hodnotu, nemůže agregovaný kořen zaručit jeho invariantní, jeho platnost nebo konzistenci.</span><span class="sxs-lookup"><span data-stu-id="12501-155">If you go around the aggregate root, the aggregate root cannot guarantee its invariants, its validity, or its consistency.</span></span> <span data-ttu-id="12501-156">Nakonec budete mít Spaghetti kód nebo transakční kód skriptu.</span><span class="sxs-lookup"><span data-stu-id="12501-156">Eventually you will have spaghetti code or transactional script code.</span></span>

<span data-ttu-id="12501-157">Aby bylo možné sledovat vzory DDD, nesmí entity mít veřejné metody setter v žádné vlastnosti entity.</span><span class="sxs-lookup"><span data-stu-id="12501-157">To follow DDD patterns, entities must not have public setters in any entity property.</span></span> <span data-ttu-id="12501-158">Změny v entitě by měly být ovládány explicitními metodami s explicitním všudypřítomným jazykem o změně, kterou provádějí v entitě.</span><span class="sxs-lookup"><span data-stu-id="12501-158">Changes in an entity should be driven by explicit methods with explicit ubiquitous language about the change they are performing in the entity.</span></span>
