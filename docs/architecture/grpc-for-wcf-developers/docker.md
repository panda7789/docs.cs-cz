## <a name="create-a-docker-image"></a><span data-ttu-id="ef466-135">Vytvoření image Docker</span><span class="sxs-lookup"><span data-stu-id="ef466-135">Create a Docker image</span></span>

<span data-ttu-id="ef466-136">Image Docker je definovaná pomocí *souboru Dockerfile*, což je textový soubor, který obsahuje všechny příkazy potřebné k sestavení aplikace a instalaci všech závislostí, které jsou potřebné pro vytvoření nebo spuštění aplikace.</span><span class="sxs-lookup"><span data-stu-id="ef466-136">A Docker image is defined by a *Dockerfile*, a text file that contains all the commands that are needed to build the application and install any dependencies that are required for either building or running the application.</span></span> <span data-ttu-id="ef466-137">Následující příklad ukazuje nejjednodušší souboru Dockerfile pro aplikaci ASP.NET Core 3,0:</span><span class="sxs-lookup"><span data-stu-id="ef466-137">The following example shows the simplest Dockerfile for an ASP.NET Core 3.0 application:</span></span>

```dockerfile

WORKDIR /src

COPY . .

RUN dotnet restore

RUN dotnet publish -c Release -o /published src/StockData/StockData.csproj

# Runtime image creation
FROM mcr.microsoft.com/dotnet/core/aspnet:3.0
```

<span data-ttu-id="ef466-138">Souboru Dockerfile má dvě části: první používá základní image `sdk` k sestavení a publikování aplikace. <span data-ttu-id="ef466-180">Pokud chcete přejít do dokovacího centra, pojmenujte název image názvem uživatele nebo organizace.</span><span class="sxs-lookup"><span data-stu-id="ef466-180">To push to the Docker Hub, prefix the image name with your user or organization name.</span></span>

```console
docker tag stockdata myorg/stockdata
docker push myorg/stockdata
```

<span data-ttu-id="ef466-181">Pokud chcete přejít do privátního registru, použijte předponu názvu bitové kopie s názvem hostitele registru a názvem organizace.</span><span class="sxs-lookup"><span data-stu-id="ef466-181">To push to a private registry, prefix the image name with the registry host name and the organization name.</span></span>


<span data-ttu-id="ef466-182">Jakmile je obrázek v registru, můžete ho nasadit na jednotlivé hostitele Docker nebo na modul pro orchestraci kontejnerů, jako je Kubernetes.</span><span class="sxs-lookup"><span data-stu-id="ef466-182">Once the image is in a registry, you can deploy it to individual Docker hosts or to a container orchestration engine like Kubernetes.</span></span>
