<span data-ttu-id="5d34e-106">**Image kontejneru**: Balíček se všemi závislostmi a informacemi potřebnými k vytvoření kontejneru.</span><span class="sxs-lookup"><span data-stu-id="5d34e-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="5d34e-107">Bitová kopie obsahuje všechny závislosti (například architektury) plus konfigurace nasazení a spuštění, které mají být použity v době runtime kontejneru.</span><span class="sxs-lookup"><span data-stu-id="5d34e-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="5d34e-108">Obraz obvykle pochází z více základních obrazů, které jsou vrstvy naskládané nad sebou a tvoří souborový systém kontejneru.</span><span class="sxs-lookup"><span data-stu-id="5d34e-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container's filesystem.</span></span> <span data-ttu-id="5d34e-109">Obrázek je neměnný, jakmile byl vytvořen.</span><span class="sxs-lookup"><span data-stu-id="5d34e-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="5d34e-110">**Dockerfile**: Textový soubor, který obsahuje pokyny, jak vytvořit image Dockeru.</span><span class="sxs-lookup"><span data-stu-id="5d34e-110">**Dockerfile**: A text file that contains instructions on how to build a Docker image.</span></span> <span data-ttu-id="5d34e-111">Je to jako dávkový skript, první řádek uvádí základní bitovou kopii a pak postupujte podle pokynů k instalaci požadovaných programů, kopírování souborů a tak dále, dokud nezískáte pracovní prostředí, které potřebujete.</span><span class="sxs-lookup"><span data-stu-id="5d34e-111">It's like a batch script, the first line states the base image to begin with and then follow the instructions to install required programs, copy files, and so on, until you get the working environment you need.</span></span>

<span data-ttu-id="5d34e-112">**Sestavení**: Akce vytváření image kontejneru na základě informací a kontextu poskytované jeho Dockerfile, plus další soubory ve složce, kde je vytvořena image.</span><span class="sxs-lookup"><span data-stu-id="5d34e-112">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="5d34e-113">Můžete vytvářet image **`docker build`** pomocí příkazu Docker.</span><span class="sxs-lookup"><span data-stu-id="5d34e-113">You can build images with the Docker **`docker build`** command.</span></span>

<span data-ttu-id="5d34e-114">**Kontejner**: Instance image Dockeru.</span><span class="sxs-lookup"><span data-stu-id="5d34e-114">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="5d34e-115">Kontejner představuje spuštění jedné aplikace, procesu nebo služby.</span><span class="sxs-lookup"><span data-stu-id="5d34e-115">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="5d34e-116">Skládá se z obsahu image Dockeru, prostředí spuštění a standardní sady instrukcí.</span><span class="sxs-lookup"><span data-stu-id="5d34e-116">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="5d34e-117">Při škálování služby vytvoříte více instancí kontejneru ze stejné bitové kopie.</span><span class="sxs-lookup"><span data-stu-id="5d34e-117">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="5d34e-118">Nebo dávková úloha můžete vytvořit více kontejnerů ze stejného obrázku, předávání různých parametrů pro každou instanci.</span><span class="sxs-lookup"><span data-stu-id="5d34e-118">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="5d34e-119">**Svazky**: Nabídněte zapisovatelný souborový systém, který může kontejner používat.</span><span class="sxs-lookup"><span data-stu-id="5d34e-119">**Volumes**: Offer a writable filesystem that the container can use.</span></span> <span data-ttu-id="5d34e-120">Vzhledem k tomu, že obrázky jsou jen pro čtení, ale většina programů musí zapisovat do souborového systému, svazky přidávají zapisovatelnou vrstvu nad obraz kontejneru, takže programy mají přístup k zapisovatelnému souborovému systému.</span><span class="sxs-lookup"><span data-stu-id="5d34e-120">Since images are read-only but most programs need to write to the filesystem, volumes add a writable layer, on top of the container image, so the programs have access to a writable filesystem.</span></span> <span data-ttu-id="5d34e-121">Program neví, že přistupuje k vrstvené souborové soustavě, je to jen souborový systém jako obvykle.</span><span class="sxs-lookup"><span data-stu-id="5d34e-121">The program doesn't know it's accessing a layered filesystem, it's just the filesystem as usual.</span></span> <span data-ttu-id="5d34e-122">Svazky jsou aktivní v hostitelském systému a jsou spravovány Dockerem.</span><span class="sxs-lookup"><span data-stu-id="5d34e-122">Volumes live in the host system and are managed by Docker.</span></span>

<span data-ttu-id="5d34e-123">**Značka**: Značka nebo popisek, který můžete použít na obrázky, aby bylo možné identifikovat různé obrázky nebo verze stejného obrázku (v závislosti na čísle verze nebo cílovém prostředí).</span><span class="sxs-lookup"><span data-stu-id="5d34e-123">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="5d34e-124">**Vícestupňové sestavení**: Je funkce, protože Docker 17.05 nebo vyšší, který pomáhá snížit velikost konečné image.</span><span class="sxs-lookup"><span data-stu-id="5d34e-124">**Multi-stage Build**: Is a feature, since Docker 17.05 or higher, that helps to reduce the size of the final images.</span></span> <span data-ttu-id="5d34e-125">V několika větách, s vícestupňovým sestavením můžete použít například velkou základní bitovou kopii obsahující sdk, pro kompilaci a publikování aplikace a následné použití složky publikování s malou základní bitovou kopii pouze za běhu, abyste vytvořili mnohem menší konečnou bitovou kopii</span><span class="sxs-lookup"><span data-stu-id="5d34e-125">In a few sentences, with multi-stage build you can use, for example, a large base image, containing the SDK, for compiling and publishing the application and then using the publishing folder with a small runtime-only base image, to produce a much smaller final image</span></span>

<span data-ttu-id="5d34e-126">**Úložiště (úložiště)**: Kolekce souvisejících imitek Dockeru, označená značkou, která označuje verzi bitové kopie.</span><span class="sxs-lookup"><span data-stu-id="5d34e-126">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="5d34e-127">Některá repo repo obsahují více variant určitého obrázku, například obrázek obsahující sady SDK (těžší), obrázek obsahující pouze runtimes (světlejší) atd. Tyto varianty mohou být označeny značkami.</span><span class="sxs-lookup"><span data-stu-id="5d34e-127">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="5d34e-128">Jediné repo může obsahovat varianty platformy, jako je například bitová kopie Linuxu a bitová kopie systému Windows.</span><span class="sxs-lookup"><span data-stu-id="5d34e-128">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="5d34e-129">**Registr**: Služba, která poskytuje přístup k úložištím.</span><span class="sxs-lookup"><span data-stu-id="5d34e-129">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="5d34e-130">Výchozí registr pro většinu veřejných bitových kopií je [Docker Hub](https://hub.docker.com/) (vlastněný Dockerem jako organizace).</span><span class="sxs-lookup"><span data-stu-id="5d34e-130">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="5d34e-131">Registr obvykle obsahuje úložiště z více týmů.</span><span class="sxs-lookup"><span data-stu-id="5d34e-131">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="5d34e-132">Společnosti mají často soukromé registry pro ukládání a správu obrázků, které vytvořily.</span><span class="sxs-lookup"><span data-stu-id="5d34e-132">Companies often have private registries to store and manage images they've created.</span></span> <span data-ttu-id="5d34e-133">Dalším příkladem je Azure Container Registry.</span><span class="sxs-lookup"><span data-stu-id="5d34e-133">Azure Container Registry is another example.</span></span>

<span data-ttu-id="5d34e-134">**Víceobloukový obraz**: Pro vícearchitekturje funkce, která zjednodušuje výběr příslušné bitové kopie podle platformy, na které je **`FROM mcr.microsoft.com/dotnet/core/sdk:2.2`** spuštěný Docker, **`2.2-nanoserver-1709`** například když soubor Dockerfile požaduje základní bitovou kopii z registru, který skutečně získá , **`2.2-nanoserver-1803`** **`2.2-nanoserver-1809`** nebo **`2.2-stretch`**, v závislosti na operačním systému a verzi, kde je docker spuštěn.</span><span class="sxs-lookup"><span data-stu-id="5d34e-134">**Multi-arch image**: For multi-architecture, is a feature that simplifies the selection of the appropriate image, according to the platform where Docker is running, for example, when a Dockerfile requests a base image **`FROM mcr.microsoft.com/dotnet/core/sdk:2.2`** from the registry it actually gets **`2.2-nanoserver-1709`**, **`2.2-nanoserver-1803`**, **`2.2-nanoserver-1809`** or **`2.2-stretch`**, depending on the operating system and version where Docker is running.</span></span>

<span data-ttu-id="5d34e-135">**Docker Hub**: Veřejný registr pro nahrávání obrázků a práci s nimi.</span><span class="sxs-lookup"><span data-stu-id="5d34e-135">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="5d34e-136">Docker Hub poskytuje hostování image Dockeru, veřejné nebo soukromé registry, aktivační události sestavení a webové háčky a integraci s GitHuba a Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="5d34e-136">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="5d34e-137">**Azure Container Registry**: Veřejný prostředek pro práci s ibi Dockeru a jeho součástmi v Azure.</span><span class="sxs-lookup"><span data-stu-id="5d34e-137">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="5d34e-138">To poskytuje registr, který je blízko k vašim nasazením v Azure a který vám dává kontrolu nad přístupem, takže je možné používat skupiny a oprávnění Služby Azure Active Directory.</span><span class="sxs-lookup"><span data-stu-id="5d34e-138">This provides a registry that's close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="5d34e-139">**Docker Trusted Registry (DTR)**: Služba registru Dockeru (z Dockeru), kterou lze nainstalovat místně, aby se stěžejně v datovém centru a síti organizace.</span><span class="sxs-lookup"><span data-stu-id="5d34e-139">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization's datacenter and network.</span></span> <span data-ttu-id="5d34e-140">Je vhodný pro soukromé obrázky, které by měly být spravovány v rámci podniku.</span><span class="sxs-lookup"><span data-stu-id="5d34e-140">It's convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="5d34e-141">Důvěryhodný registr Dockeru je součástí produktu Docker Datacenter.</span><span class="sxs-lookup"><span data-stu-id="5d34e-141">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="5d34e-142">Další informace naleznete v [tématu Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="5d34e-142">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="5d34e-143">**Docker Community Edition (CE):** Vývojové nástroje pro Windows a macOS pro místní vytváření, spouštění a testování kontejnerů.</span><span class="sxs-lookup"><span data-stu-id="5d34e-143">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="5d34e-144">Docker CE pro Windows poskytuje vývojová prostředí pro linuxové i windowsové kontejnery.</span><span class="sxs-lookup"><span data-stu-id="5d34e-144">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="5d34e-145">Linux Docker host v systému Windows je založen na virtuálním počítači [Hyper-V.](https://www.microsoft.com/cloud-platform/server-virtualization)</span><span class="sxs-lookup"><span data-stu-id="5d34e-145">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) virtual machine.</span></span> <span data-ttu-id="5d34e-146">Hostitel pro kontejnery systému Windows je přímo založen na systému Windows.</span><span class="sxs-lookup"><span data-stu-id="5d34e-146">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="5d34e-147">Docker CE for Mac je založen na rozhraní Apple Hypervisor a [hypervisoru xhyve](https://github.com/mist64/xhyve), který poskytuje virtuální počítač hostitele Linux Docker u Mac OS X. <span data-ttu-id="5d34e-148">**Docker Enterprise Edition (EE):** Verze nástrojů Dockeru v podnikovém měřítku pro vývoj Linuxu a Windows.</span><span class="sxs-lookup"><span data-stu-id="5d34e-148">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="5d34e-149">**Compose**: Nástroj příkazového řádku a formát souboru YAML s metadaty pro definování a spouštění aplikací s více kontejnery.</span><span class="sxs-lookup"><span data-stu-id="5d34e-149">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="5d34e-150">Definujete jednu aplikaci založenou na více obrazech s jedním nebo více soubory .yml, které mohou přepsat hodnoty v závislosti na prostředí.</span><span class="sxs-lookup"><span data-stu-id="5d34e-150">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="5d34e-151">Po vytvoření definic můžete nasadit celou aplikaci s více kontejnery pomocí jediného příkazu (docker-compose up), který vytvoří kontejner na bitovou kopii na hostiteli Dockeru.</span><span class="sxs-lookup"><span data-stu-id="5d34e-151">After you've created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>
