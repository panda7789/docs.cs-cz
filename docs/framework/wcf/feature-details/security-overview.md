### <a name="transfer-security"></a><span data-ttu-id="963a1-172">Přenos zabezpečení</span><span class="sxs-lookup"><span data-stu-id="963a1-172">Transfer Security</span></span>  
 <span data-ttu-id="963a1-173">Přenos zabezpečení zahrnuje tři hlavní funkce zabezpečení: integrita, důvěrnost a ověřování.</span><span class="sxs-lookup"><span data-stu-id="963a1-173">Transfer security encompasses three major security functions: integrity, confidentiality, and authentication.</span></span> <span data-ttu-id="963a1-174">*Integrita* je schopnost zjistit, zda byla zpráva úmyslně poškozena.</span><span class="sxs-lookup"><span data-stu-id="963a1-174">*Integrity* is the ability to detect whether a message has been tampered with.</span></span> <span data-ttu-id="963a1-175">*Důvěrné* je schopnost uchovávat zprávu, kterou nemůže číst nikdo jiný než zamýšlený příjemce; toho je dosaženo prostřednictvím kryptografie.</span><span class="sxs-lookup"><span data-stu-id="963a1-175">*Confidentiality* is the ability to keep a message unreadable by anyone other than the intended recipient; this is achieved through cryptography.</span></span> <span data-ttu-id="963a1-176">*Ověřování* je schopnost ověřit deklarované identity.</span><span class="sxs-lookup"><span data-stu-id="963a1-176">*Authentication* is the ability to verify a claimed identity.</span></span> <span data-ttu-id="963a1-177">Společně tyto tři funkce vám pomůžou zajistit, aby zprávy byly bezpečně doručeny z jednoho bodu do druhého.</span><span class="sxs-lookup"><span data-stu-id="963a1-177">Together, these three functions help to ensure that messages securely arrive from one point to another.</span></span>  
  
#### <a name="transport-and-message-security-modes"></a><span data-ttu-id="963a1-178">Režimy zabezpečení přenosů a zpráv</span><span class="sxs-lookup"><span data-stu-id="963a1-178">Transport and Message Security Modes</span></span>  
 <span data-ttu-id="963a1-179">K implementaci přenosu zabezpečení ve službě WCF se používají dva hlavní mechanismy: režim zabezpečení *přenosu* a režim zabezpečení *zpráv* .</span><span class="sxs-lookup"><span data-stu-id="963a1-179">Two main mechanisms are used to implement transfer security in WCF: *transport* security mode and *message* security mode.</span></span>  
  
- <span data-ttu-id="963a1-180">*Režim zabezpečení přenosu* používá k zajištění zabezpečení přenosu protokol na úrovni přenosu, jako je například https.</span><span class="sxs-lookup"><span data-stu-id="963a1-180">*Transport security mode* uses a transport-level protocol, such as HTTPS, to achieve transfer security.</span></span> <span data-ttu-id="963a1-181">Přenosový režim má výhodu širokého přijetí, k dispozici na mnoha platformách a méně výpočetně složitých.</span><span class="sxs-lookup"><span data-stu-id="963a1-181">Transport mode has the advantage of being widely adopted, available on many platforms, and less computationally complex.</span></span> <span data-ttu-id="963a1-182">Nicméně má nevýhodu zabezpečení zpráv pouze z typu Point-to-Point.</span><span class="sxs-lookup"><span data-stu-id="963a1-182">However, it has the disadvantage of securing messages only from point-to-point.</span></span>  
  
- <span data-ttu-id="963a1-183">*Režim zabezpečení zpráv*na druhé straně používá k implementaci přenosu zabezpečení protokol WS-Security (a další specifikace).</span><span class="sxs-lookup"><span data-stu-id="963a1-183">*Message security mode*, on the other hand, uses WS-Security (and other specifications) to implement transfer security.</span></span> <span data-ttu-id="963a1-184">Vzhledem k tomu, že se zabezpečení zpráv používá přímo na zprávy protokolu SOAP a je obsaženo uvnitř obálek protokolu SOAP spolu s daty aplikace, má výhodu přenosů, které jsou nezávislé na přenosu, rozšiřitelnosti a zajištění komplexního zabezpečení (oproti Point-to-Point); Nevýhodou je více pomalejší než režim zabezpečení přenosu, protože se musí zabývat XML povahou zpráv SOAP.</span><span class="sxs-lookup"><span data-stu-id="963a1-184">Because the message security is applied directly to the SOAP messages and is contained inside the SOAP envelopes, together with the application data, it has the advantage of being transport protocol-independent, more extensible, and ensuring end-to-end security (versus point-to-point); it has the disadvantage of being several times slower than transport security mode because it has to deal with the XML nature of the SOAP messages.</span></span>  
  
 <span data-ttu-id="963a1-185">Další informace o těchto rozdílech najdete v tématu [zabezpečení služeb a klientů](securing-services-and-clients.md).</span><span class="sxs-lookup"><span data-stu-id="963a1-185">For more information about these differences, see [Securing Services and Clients](securing-services-and-clients.md).</span></span>  
  
 <span data-ttu-id="963a1-186">Třetí režim zabezpečení používá předchozí režimy a přináší výhody obou.</span><span class="sxs-lookup"><span data-stu-id="963a1-186">A third security mode uses both previous modes and brings advantages of both.</span></span> <span data-ttu-id="963a1-187">Tento režim je volán `TransportWithMessageCredential` .</span><span class="sxs-lookup"><span data-stu-id="963a1-187">This mode is called `TransportWithMessageCredential`.</span></span> <span data-ttu-id="963a1-188">V tomto režimu se k ověřování serveru používá zabezpečení zprávy a zabezpečení přenosu se používá k ověření serveru a zajištění důvěrnosti a integrity zpráv.</span><span class="sxs-lookup"><span data-stu-id="963a1-188">In this mode, message security is used to authenticate the client and transport security is used to authenticate the server and provide message confidentiality and integrity.</span></span> <span data-ttu-id="963a1-189">Díky tomu `TransportWithMessageCredential` je režim zabezpečení skoro stejně rychlý jako režim zabezpečení přenosu a poskytuje rozšiřitelné ověřování klientů stejným způsobem jako zabezpečení zpráv.</span><span class="sxs-lookup"><span data-stu-id="963a1-189">Thanks to this, the `TransportWithMessageCredential` security mode is almost as fast as transport security mode and provides client authentication extensibility in the same way as message security.</span></span> <span data-ttu-id="963a1-190">Na rozdíl od režimu zabezpečení zpráv ale neposkytuje kompletní zabezpečení.</span><span class="sxs-lookup"><span data-stu-id="963a1-190">However, unlike message security mode, it does not provide complete end-to-end security.</span></span>  
  
### <a name="access-control"></a><span data-ttu-id="963a1-191">Řízení přístupu</span><span class="sxs-lookup"><span data-stu-id="963a1-191">Access Control</span></span>  
 <span data-ttu-id="963a1-192">*Řízení přístupu* se také označuje jako autorizace.</span><span class="sxs-lookup"><span data-stu-id="963a1-192">*Access control* is also known as authorization.</span></span> <span data-ttu-id="963a1-193">*Autorizace* umožňuje různým uživatelům mít různá oprávnění k zobrazení dat.</span><span class="sxs-lookup"><span data-stu-id="963a1-193">*Authorization* allows different users to have different privileges to view data.</span></span> <span data-ttu-id="963a1-194">Například protože soubory lidských zdrojů společnosti obsahují citlivá data zaměstnanců, smějí zobrazovat data zaměstnanců pouze manažeři.</span><span class="sxs-lookup"><span data-stu-id="963a1-194">For example, because a company's human resources files contain sensitive employee data, only managers are allowed to view employee data.</span></span> <span data-ttu-id="963a1-195">Kromě toho mohou manažeři zobrazit pouze data pro své přímé sestavy.</span><span class="sxs-lookup"><span data-stu-id="963a1-195">Further, managers can view only data for their direct reports.</span></span> <span data-ttu-id="963a1-196">V tomto případě je řízení přístupu založeno jak na roli ("správce"), tak i na konkrétní identitě správce (aby se zabránilo jednomu manažerovi v prohlížení záznamů zaměstnanců jiného manažera).</span><span class="sxs-lookup"><span data-stu-id="963a1-196">In this case, access control is based on both the role ("manager") as well as the specific identity of the manager (to prevent one manager from looking at another manager's employee records).</span></span>  
  
 <span data-ttu-id="963a1-197">Ve službě WCF jsou funkce řízení přístupu poskytovány prostřednictvím integrace s modulem CLR (Common Language Runtime) <xref:System.Security.Permissions.PrincipalPermissionAttribute> a sadou rozhraní API, která se označují jako *model identity*.</span><span class="sxs-lookup"><span data-stu-id="963a1-197">In WCF, access control features are provided through integration with the common language runtime (CLR) <xref:System.Security.Permissions.PrincipalPermissionAttribute> and through a set of APIs known as the *identity model*.</span></span> <span data-ttu-id="963a1-198">Podrobnosti o řízení přístupu a autorizaci na základě deklarací najdete v tématu [rozšíření zabezpečení](../extending/extending-security.md).</span><span class="sxs-lookup"><span data-stu-id="963a1-198">For details about access control and claims-based authorization, see [Extending Security](../extending/extending-security.md).</span></span>
