<span data-ttu-id="deaa9-103">Funkce Windows Communication Foundation (WCF) je schopnost vytvářet zabezpečené relace mezi dvěma koncovými body, které jsou vzájemně ověřovány, a souhlasím s procesem šifrování a digitálního podpisu.</span><span class="sxs-lookup"><span data-stu-id="deaa9-103">A feature of Windows Communication Foundation (WCF) is the ability to establish secure sessions between two endpoints that authenticate each other and agree upon an encryption and digital signature process.</span></span> <span data-ttu-id="deaa9-104">Koncový bod služby může například vyžadovat, aby koncový bod klienta odesílal token zabezpečení založený na certifikátu X. 509 pro ověřování.</span><span class="sxs-lookup"><span data-stu-id="deaa9-104">For example, the service endpoint might require a client endpoint to send a security token based upon an X.509 certificate for authentication.</span></span> <span data-ttu-id="deaa9-105">Po ověření klienta vrátí koncový bod služby token kontextu zabezpečení (SCT) zpátky klientovi, který se pak použije k zabezpečení všech následných zpráv v rámci relace.</span><span class="sxs-lookup"><span data-stu-id="deaa9-105">Once the client is data-ttu-id="deaa9-106">Vytvořením této zabezpečené relace povolíte efektivnější vyměňování sady zpráv mezi dvěma koncovými body, protože SCT má symetrický klíč.</span><span class="sxs-lookup"><span data-stu-id="deaa9-106">Establishing this secure session enables the set of messages that are exchanged between the two endpoints to be more efficient, because the SCT has a symmetric key.</span></span> <span data-ttu-id="deaa9-107">Asymetrické klíče, na kterých jsou založeny certifikáty X. 509, vyžadují při generování digitálního podpisu nebo šifrování sady dat výrazně větší výpočetní výkon než symetrický klíč.</span><span class="sxs-lookup"><span data-stu-id="deaa9-107">Asymmetric keys, which X.509 certificates are based upon, require significantly more computational  <span data-ttu-id="deaa9-108">Zásada Bootstrap (definovaná v části 6.2.7 standardu [WS-SecurityPolicy](https://docs.oasis-open.org/ws-sx/ws-securitypolicy/200702/ws-securitypolicy-1.2-spec-os.html) ) obsahuje kontrolní výrazy zabezpečení zprávy použité k zabezpečení kanálu a ověřování klienta před protokolem RST/SCT a RSTR/SCT Exchange.</span><span class="sxs-lookup"><span data-stu-id="deaa9-108">The bootstrap policy (defined in section 6.2.7 of the [WS-SecurityPolicy](https://docs.oasis-open.org/ws-sx/ws-securitypolicy/200702/ws-securitypolicy-1.2-spec-os.html) standard) contains the message security assertions used to secure the channel and authenticate the client prior to the RST/SCT and RSTR/SCT exchange.</span></span> <span data-ttu-id="deaa9-109">Některé standardní vazby WCF mají vlastnost `Security.Message.EstablishSecurityContext`, která určuje, jestli se používá zabezpečená konverzace.</span><span class="sxs-lookup"><span data-stu-id="deaa9-109">Certain WCF standard bindings have a `Security.Message.EstablishSecurityContext` property which controls whether secure conversation is used.</span></span> <span data-ttu-id="deaa9-110">Při použití vlastních vazeb je Bootstrap uvedeno vnořením prvků zabezpečení vazby, a to buď pomocí [\<secureConversationBootstrap >](../../../../docs/framework/configure-apps/file-schema/wcf/secureconversationbootstrap.md) v konfiguračním souboru, nebo voláním <xref:System.ServiceModel.Channels.SecurityBindingElement.CreateSecureConversationBindingElement%2A> v kódu.</span><span class="sxs-lookup"><span data-stu-id="deaa9-110">When using custom bindings the bootstrap is indicated by nesting security binding elements, either through [\<secureConversationBootstrap>](../../../../docs/framework/configure-apps/file-schema/wcf/secureconversationbootstrap.md) in the configuration file, or by calling <xref:System.ServiceModel.Channels.SecurityBindingElement.CreateSecureConversationBindingElement%2A> in code.</span></span>
