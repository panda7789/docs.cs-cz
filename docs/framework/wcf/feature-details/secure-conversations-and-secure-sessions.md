<span data-ttu-id="3c48a-103">Funkce Windows Communication Foundation (WCF) je schopnost vytvářet zabezpečené relace mezi dvěma koncovými body, které jsou vzájemně ověřovány, a souhlasím s procesem šifrování a digitálního podpisu.</span><span class="sxs-lookup"><span data-stu-id="3c48a-103">A feature of Windows Communication Foundation (WCF) is the ability to establish secure sessions between two endpoints that authenticate each other and agree upon an encryption and digital signature process.</span></span> <span data-ttu-id="3c48a-104">Koncový bod služby může například vyžadovat, aby koncový bod klienta odesílal token zabezpečení založený na certifikátu X. 509 pro ověřování.</span><span class="sxs-lookup"><span data-stu-id="3c48a-104">For example, the service endpoint might require a client endpoint to send a security token based upon an X.509 certificate for authentication.</span></span> <span data-ttu-id="3c48a-105">Po ověření klienta vrátí koncový bod služby token kontextu zabezpečení (SCT) zpátky klientovi, který se pak použije k zabezpečení všech následných zpráv v rámci relace.</span><span class="sxs-lookup"><span data-stu-id="3c48a-105">Once the client is authenticated, the service endpoint returns a security context token (SCT) back to the client that is then used to secure all subsequent messages within the session.</span></span> <span data-ttu-id="3c48a-106">Vytvořením této zabezpečené relace povolíte efektivnější vyměňování sady zpráv mezi dvěma koncovými body, protože SCT má symetrický klíč.</span><span class="sxs-lookup"><span data-stu-id="3c48a-106">Establishing this secure session enables the set of messages that are exchanged between the two endpoints to be more efficient, because the SCT has a symmetric key.</span></span> <span data-ttu-id="3c48a-107">Asymetrické klíče, na kterých jsou založeny certifikáty X. 509, vyžadují při generování digitálního podpisu nebo šifrování sady dat výrazně větší výpočetní výkon než symetrický klíč.</span><span class="sxs-lookup"><span data-stu-id="3c48a-107">Asymmetric keys, which X.509 certificates are based upon, require significantly more computational power than symmetric keys when generating a digital signature or encrypting a set of data.</span></span>  
  
 <span data-ttu-id="3c48a-108">Zásada Bootstrap (definovaná v části 6.2.7 standardu [WS-SecurityPolicy](https://docs.oasis-open.org/ws-sx/ws-securitypolicy/200702/ws-securitypolicy-1.2-spec-os.html) ) obsahuje kontrolní výrazy zabezpečení zprávy použité k zabezpečení kanálu a ověřování klienta před protokolem RST/SCT a RSTR/SCT Exchange.</span><span class="sxs-lookup"><span data-stu-id="3c48a-108">The bootstrap policy (defined in section 6.2.7 of the [WS-SecurityPolicy](https://docs.oasis-open.org/ws-sx/ws-securitypolicy/200702/ws-securitypolicy-1.2-spec-os.html) standard) contains the message security assertions used to secure the channel and authenticate the client prior to the RST/SCT and RSTR/SCT exchange.</span></span> <span data-ttu-id="3c48a-109">Některé standardní vazby WCF mají `Security.Message.EstablishSecurityContext` vlastnost, která určuje, jestli se používá zabezpečená konverzace.</span><span class="sxs-lookup"><span data-stu-id="3c48a-109">Certain WCF standard bindings have a `Security.Message.EstablishSecurityContext` property which controls whether secure conversation is used.</span></span> <span data-ttu-id="3c48a-110">Při použití vlastních vazeb je Bootstrap uvedeno vnořením prvků zabezpečení vazby, a to buď [\<secureConversationBootstrap>](../../configure-apps/file-schema/wcf/secureconversationbootstrap.md) do konfiguračního souboru, nebo voláním <xref:System.ServiceModel.Channels.SecurityBindingElement.CreateSecureConversationBindingElement%2A> v kódu.</span><span class="sxs-lookup"><span data-stu-id="3c48a-110">When using custom bindings the bootstrap is indicated by nesting security binding elements, either through [\<secureConversationBootstrap>](../../configure-apps/file-schema/wcf/secureconversationbootstrap.md) in the configuration file, or by calling <xref:System.ServiceModel.Channels.SecurityBindingElement.CreateSecureConversationBindingElement%2A> in code.</span></span>
