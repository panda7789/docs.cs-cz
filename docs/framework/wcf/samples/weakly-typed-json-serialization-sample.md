<span data-ttu-id="6505b-103">Při serializaci uživatelsky definovaného typu do daného formátu nebo deserializaci formátu přenosu zpět do uživatelsky definovaného typu musí být daný uživatelsky definovaný typ k dispozici jak v rámci služby, tak i v klientovi.</span><span class="sxs-lookup"><span data-stu-id="6505b-103">When serializing a user-defined type to a given wire format, or deserializing a wire format back into a user-defined type, the given user-defined type must be available on both the service and the client.</span></span> <span data-ttu-id="6505b-104">K tomu obvykle je použit atribut <xref:System.Runtime.Serialization.DataContractAttribute> pro tyto uživatelsky definované typy a atribut <xref:System.Runtime.Serialization.DataMemberAttribute> je použit na jejich členy.</span><span class="sxs-lookup"><span data-stu-id="6505b-104">Usually to accomplish this, the <xref:System.Runtime.Serialization.DataContractAttribute> attribute is applied to these user-defined types and the <xref:System.Runtime.Serialization.DataMemberAttribute> attribute is applied to their members.</span></span> <span data-ttu-id="6505b-105">Tento mechanismus platí také při práci s objekty JavaScript Object Notation (JSON), jak je popsáno v tématu [Postupy: serializace a deserializace dat JSON](../../../../docs/framework/wcf/feature-details/how-to-serialize-and-deserialize-json-data.md).</span><span class="sxs-lookup"><span data-stu-id="6505b-105">This mechanism also applies when working with JavaScript Object Notation (JSON) objects, as described in the topic [How to: Serialize and Deserialize JSON Data](../../../../docs/framework/wcf/feature-details/how-to-serialize-and-deserialize-json-data.md).</span></span>  
  
 <span data-ttu-id="6505b-106">V některých scénářích musí služba Windows Communication Foundation (WCF) nebo klient přistupovat k objektům JSON generovaným službou nebo klientem, který se nachází mimo ovládací prvek vývojáře.</span><span class="sxs-lookup"><span data-stu-id="6505b-106">In some scenarios, a Windows Communication Foundation (WCF) service or client must access JSON objects generated by a service or client that is outside of the control of the developer.</span></span> <span data-ttu-id="6505b-107">V případě, že více webových služeb zveřejňuje rozhraní API JSON, může být pro vývojáře WCF nepraktické vytvořit místní uživatelsky definované typy, do kterých se mají deserializovat libovolné objekty JSON.</span><span class="sxs-lookup"><span data-stu-id="6505b-107">As more Web services publicly expose JSON APIs, it can become impractical for the WCF developer to construct local user-defined types into which to deserialize arbitrary JSON objects.</span></span> <span data-ttu-id="6505b-108">Tato ukázka poskytuje mechanismus, který vývojářům WCF umožňuje pracovat s deserializovanými a libovolnými objekty JSON, aniž by museli vytvářet uživatelsky definované typy.</span><span class="sxs-lookup"><span data-stu-id="6505b-108">This sample provides a mechanism that enables WCF developers to work with deserialized, arbitrary JSON objects, without creating user-defined types.</span></span> <span data-ttu-id="6505b-109">To se označuje jako *slabě typované serializace* objektů JSON, protože typ, ve kterém je deserializace objektu JSON, není v době kompilace znám.</span><span class="sxs-lookup"><span data-stu-id="6505b-109">This is known as *weakly-typed serialization* of JSON objects, because the type into which a JSON object deserializes is not known at compile time.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="6505b-110">Postup nastavení a pokyny pro sestavení pro tuto ukázku najdete na konci tohoto tématu.</span><span class="sxs-lookup"><span data-stu-id="6505b-110">The setup procedure and build instructions for this sample are located at the end of this topic.</span></span>  
  
 <span data-ttu-id="6505b-111">Například rozhraní API veřejné webové služby vrací následující objekt JSON, který popisuje některé informace o uživateli služby.</span><span class="sxs-lookup"><span data-stu-id="6505b-111">For example, a public Web service API returns the following JSON object, which describes some information about a user of the service.</span></span>  
  
```json  
{"personal": {"name": "Paul", "age": 23, "height": 1.7, "isSingle": true, "luckyNumbers": [5,17,21]}, "favoriteBands": ["Band ABC", "Band XYZ"]}  
```  
  
 <span data-ttu-id="6505b-112">K deserializaci tohoto objektu musí klient WCF implementovat následující uživatelsky definované typy.</span><span class="sxs-lookup"><span data-stu-id="6505b-112">To deserialize this object, a WCF client must implement the following user-defined types.</span></span>  
  
```csharp

 public class MemberProfile  
 {  
     [DataMember]  
     public PersonalInfo personal;  
  
     [DataMember]  
     public string[] favoriteBands;  
 }  
  
 [DataContract]  
 public class PersonalInfo  
 {  
     [DataMember]  
     public string name;  
  
     [DataMember]  
     public int age;  
  
     [DataMember]  
     public double height;  
  
     [DataMember]  
     public bool isSingle;  
  
     [DataMember]  
     public int[] luckyNumbers;  
 }  
```  
  
 <span data-ttu-id="6505b-113">To může být náročné, zejména v případě, že klient musí zpracovat více než jeden typ objektu JSON.</span><span class="sxs-lookup"><span data-stu-id="6505b-113">This can be cumbersome, especially if the client has to handle more than one type of JSON object.</span></span>  
  
 <span data-ttu-id="6505b-114">Typ `JsonObject` poskytnutý touto ukázkou zavádí slabě typové reprezentace deserializovaného objektu JSON.</span><span class="sxs-lookup"><span data-stu-id="6505b-114">The `JsonObject` type provided by this sample introduces a weakly-typed representation of the deserialized JSON object.</span></span> <span data-ttu-id="6505b-115">`JsonObject` spoléhá na přirozené mapování mezi objekty JSON a .NET Frameworkmi slovníky a mapování mezi poli JSON a .NET Framework poli.</span><span class="sxs-lookup"><span data-stu-id="6505b-115">`JsonObject` relies on the natural mapping between JSON objects and .NET Framework dictionaries, and the mapping between JSON arrays and .NET Framework arrays.</span></span> <span data-ttu-id="6505b-116">Následující kód ukazuje typ `JsonObject`.</span><span class="sxs-lookup"><span data-stu-id="6505b-116">The following code shows the `JsonObject` type.</span></span>  
  
```csharp  
// Instantiation of JsonObject json omitted  
  
string name = json["root"]["personal"]["name"];  
int age = json["root"]["personal"]["age"];  
double height = json["root"]["personal"]["height"];  
bool isSingle = json["root"]["personal"]["isSingle"];  
int[] luckyNumbers = {  
                                     json["root"]["personal"]["luckyNumbers"][0],  
                                     json["root"]["personal"]["luckyNumbers"][1],  
                                     json["root"]["personal"]["luckyNumbers"][2]   
                                 };  
string[] favoriteBands = {  
                                        json["root"]["favoriteBands"][0],  
                                        json["root"]["favoriteBands"][1]  
                                    };  
```  
  
 <span data-ttu-id="6505b-117">Všimněte si, že můžete procházet objekty JSON a pole bez nutnosti deklarovat jejich typ v době kompilace.</span><span class="sxs-lookup"><span data-stu-id="6505b-117">Note that you can "browse" JSON objects and arrays without the need to declare their type at compile time.</span></span> <span data-ttu-id="6505b-118">Vysvětlení požadavku na objekt `["root"]` nejvyšší úrovně naleznete v tématu [mapování mezi JSON a XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span><span class="sxs-lookup"><span data-stu-id="6505b-118">For an explanation of the requirement for the top-level `["root"]` object, see the topic [Mapping Between JSON and XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span></span>

    {  
        // There is no need to write a DataContract for the complex type returned by the service.

    }  
```

```  
  
 <span data-ttu-id="6505b-124">Konstruktor `JsonObject` přebírá <xref:System.Xml.XmlDictionaryReader>, který se získá prostřednictvím metody <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A>.</span><span class="sxs-lookup"><span data-stu-id="6505b-124">The `JsonObject` constructor takes a <xref:System.Xml.XmlDictionaryReader>, which is obtained through the <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> method.</span></span> <span data-ttu-id="6505b-125">Čtecí modul obsahuje reprezentaci XML zprávy JSON, kterou klient přijal.</span><span class="sxs-lookup"><span data-stu-id="6505b-125">The reader contains an XML representation of the JSON message received by the client.</span></span> <span data-ttu-id="6505b-126">Další informace naleznete v tématu [mapování mezi JSON a XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span><span class="sxs-lookup"><span data-stu-id="6505b-126">For more information, see the topic [Mapping Between JSON and XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span></span>
