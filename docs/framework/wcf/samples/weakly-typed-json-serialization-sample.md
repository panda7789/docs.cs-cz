<span data-ttu-id="7fe61-103">Při serializaci uživatelsky definovaného typu do daného formátu nebo deserializaci formátu přenosu zpět do uživatelsky definovaného typu musí být daný uživatelsky definovaný typ k dispozici jak v rámci služby, tak i v klientovi.</span><span class="sxs-lookup"><span data-stu-id="7fe61-103">When serializing a user-defined type to a given wire format, or deserializing a wire format back into a user-defined type, the given user-defined type must be available on both the service and the client.</span></span> <span data-ttu-id="7fe61-104">Obvykle k tomu je <xref:System.Runtime.Serialization.DataContractAttribute> atribut použit pro tyto uživatelsky definované typy a <xref:System.Runtime.Serialization.DataMemberAttribute> atribut je použit na své členy.</span><span class="sxs-lookup"><span data-stu-id="7fe61-104">Usually to accomplish this, the <xref:System.Runtime.Serialization.DataContractAttribute> attribute is applied to these user-defined types and the <xref:System.Runtime.Serialization.DataMemberAttribute> attribute is applied to their members.</span></span> <span data-ttu-id="7fe61-105">Tento mechanismus platí také při práci s objekty JavaScript Object Notation (JSON), jak je popsáno v tématu [Postupy: serializace a deserializace dat JSON](../feature-details/how-to-serialize-and-deserialize-json-data.md).</span><span class="sxs-lookup"><span data-stu-id="7fe61-105">This mechanism also applies when working with JavaScript Object Notation (JSON) objects, as described in the topic [How to: Serialize and Deserialize JSON Data](../feature-details/how-to-serialize-and-deserialize-json-data.md).</span></span>  
  
 <span data-ttu-id="7fe61-106">V některých scénářích musí služba Windows Communication Foundation (WCF) nebo klient přistupovat k objektům JSON generovaným službou nebo klientem, který se nachází mimo ovládací prvek vývojáře.</span><span class="sxs-lookup"><span data-stu-id="7fe61-106">In some scenarios, a Windows Communication Foundation (WCF) service or client must access JSON objects generated by a service or client that is outside of the control of the developer.</span></span> <span data-ttu-id="7fe61-107">V případě, že více webových služeb zveřejňuje rozhraní API JSON, může být pro vývojáře WCF nepraktické vytvořit místní uživatelsky definované typy, do kterých se mají deserializovat libovolné objekty JSON.</span><span class="sxs-lookup"><span data-stu-id="7fe61-107">As more Web services publicly expose JSON APIs, it can become impractical for the WCF developer to construct local user-defined types into which to deserialize arbitrary JSON objects.</span></span> <span data-ttu-id="7fe61-108">Tato ukázka poskytuje mechanismus, který vývojářům WCF umožňuje pracovat s deserializovanými a libovolnými objekty JSON, aniž by museli vytvářet uživatelsky definované typy.</span><span class="sxs-lookup"><span data-stu-id="7fe61-108">This sample provides a mechanism that enables WCF developers to work with deserialized, arbitrary JSON objects, without creating user-defined types.</span></span> <span data-ttu-id="7fe61-109">To se označuje jako *slabě typované serializace* objektů JSON, protože typ, ve kterém je deserializace objektu JSON, není v době kompilace znám.</span><span class="sxs-lookup"><span data-stu-id="7fe61-109">This is known as *weakly-typed serialization* of JSON objects, because the type into which a JSON object deserializes is not known at compile time.</span></span>

 <span data-ttu-id="7fe61-111">Například rozhraní API veřejné webové služby vrací následující objekt JSON, který popisuje některé informace o uživateli služby.</span><span class="sxs-lookup"><span data-stu-id="7fe61-111">For example, a public Web service API returns the following JSON object, which describes some information about a user of the service.</span></span>

 <span data-ttu-id="7fe61-112">K deserializaci tohoto objektu musí klient WCF implementovat následující uživatelsky definované typy.</span><span class="sxs-lookup"><span data-stu-id="7fe61-112">To deserialize this object, a WCF client must implement the following user-defined types.</span></span>

```csharp  
     [DataMember]

     [DataMember]
```  

 <span data-ttu-id="7fe61-114">`JsonObject`Typ poskytnutý touto ukázkou zavádí slabě typované reprezentace objektu JSON deserializovatelné.</span><span class="sxs-lookup"><span data-stu-id="7fe61-114">The `JsonObject` type provided by this sample introduces a weakly-typed representation of the deserialized JSON object.</span></span> <span data-ttu-id="7fe61-115">`JsonObject`spoléhá na přirozené mapování mezi objekty JSON a .NET Framework slovníku a mapování mezi poli JSON a .NET Framework poli.</span><span class="sxs-lookup"><span data-stu-id="7fe61-115">`JsonObject` relies on the natural mapping between JSON objects and .NET Framework dictionaries, and the mapping between JSON arrays and .NET Framework arrays.</span></span> <span data-ttu-id="7fe61-116">Následující kód ukazuje `JsonObject` typ.</span><span class="sxs-lookup"><span data-stu-id="7fe61-116">The following code shows the `JsonObject` type.</span></span>  
  
```csharp

                                     json["root"]["personal"]["luckyNumbers"][0],

                                    };
```  

 <span data-ttu-id="7fe61-117">Všimněte si, že můžete procházet objekty JSON a pole bez nutnosti deklarovat jejich typ v době kompilace.</span><span class="sxs-lookup"><span data-stu-id="7fe61-117">Note that you can "browse" JSON objects and arrays without the need to declare their type at compile time.</span></span> <span data-ttu-id="7fe61-118">Vysvětlení požadavku na objekt nejvyšší úrovně `["root"]` naleznete v tématu [mapování mezi JSON a XML](../feature-details/mapping-between-json-and-xml.md).</span><span class="sxs-lookup"><span data-stu-id="7fe61-118">For an explanation of the requirement for the top-level `["root"]` object, see the topic [Mapping Between JSON and XML](../feature-details/mapping-between-json-and-xml.md).</span></span>
