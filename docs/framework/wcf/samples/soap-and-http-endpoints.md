## <a name="discussion"></a><span data-ttu-id="36c6b-108">Účely</span><span class="sxs-lookup"><span data-stu-id="36c6b-108">Discussion</span></span>  
 <span data-ttu-id="36c6b-109">Tato ukázka se skládá ze dvou součástí: projektu webové aplikace (služba), který obsahuje službu WCF a konzolovou aplikaci (klient), která vyvolá operace služby pomocí vazeb SOAP a HTTP.</span><span class="sxs-lookup"><span data-stu-id="36c6b-109">This sample consists of two components: a Web Application project (Service) that contains a WCF service and a console application (Client) that invokes service operations using SOAP and HTTP bindings.</span></span>  
  
 <span data-ttu-id="36c6b-110">Služba WCF zpřístupňuje 2 operace –`GetData` a `PutData` – vyecho řetězec, který byl předán jako vstup.</span><span class="sxs-lookup"><span data-stu-id="36c6b-110">The WCF service exposes 2 operations –`GetData` and `PutData` – that echo the string that was passed as input.</span></span> <span data-ttu-id="36c6b-111">Operace služby jsou opatřeny poznámkou <xref:System.ServiceModel.Web.WebGetAttribute> a <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span><span class="sxs-lookup"><span data-stu-id="36c6b-111">The service operations are annotated with <xref:System.ServiceModel.Web.WebGetAttribute> and <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span></span> <span data-ttu-id="36c6b-112">Tyto atributy řídí projekci HTTP těchto operací.</span><span class="sxs-lookup"><span data-stu-id="36c6b-112">These attributes control the HTTP projection of these operations.</span></span> <span data-ttu-id="36c6b-113">Kromě toho jsou opatřeny poznámkami <xref:System.ServiceModel.OperationContractAttribute>, což umožňuje jejich zpřístupnění prostřednictvím vazeb SOAP.</span><span class="sxs-lookup"><span data-stu-id="36c6b-113">In addition, they are annotated with <xref:System.ServiceModel.OperationContractAttribute>, which enables them to be exposed over SOAP bindings.</span></span> <span data-ttu-id="36c6b-114">Metoda `PutData` služby vyvolá <xref:System.ServiceModel.Web.WebFaultException>, která se pošle zpátky přes HTTP pomocí kódu stavu HTTP a pošle zpátky přes SOAP jako chybu SOAP.</span><span class="sxs-lookup"><span data-stu-id="36c6b-114">The service’s `PutData` method throws a <xref:System.ServiceModel.Web.WebFaultException>, which gets sent back over HTTP using the HTTP status code and gets sent back over SOAP as a SOAP fault.</span></span>  
  
 <span data-ttu-id="36c6b-115">Soubor Web. config konfiguruje službu WCF se 3 koncovými body:</span><span class="sxs-lookup"><span data-stu-id="36c6b-115">The Web.config file configures the WCF service with 3 endpoints:</span></span>  
  
- <span data-ttu-id="36c6b-116">Koncový bod ~/Service.svc/mex, který zpřístupňuje metadata služby pro přístup klientů založených na protokolu SOAP.</span><span class="sxs-lookup"><span data-stu-id="36c6b-116">The ~/service.svc/mex endpoint that exposes the service metadata for access by SOAP-based clients.</span></span>  
  
- <span data-ttu-id="36c6b-117">Koncový bod ~/Service.svc/http, který klientům umožňuje přístup ke službě pomocí vazby HTTP.</span><span class="sxs-lookup"><span data-stu-id="36c6b-117">The ~/service.svc/http endpoint that enables clients to access the service using the HTTP binding.</span></span>  
  
- <span data-ttu-id="36c6b-118">Koncový bod ~/Service.svc/SOAP, který klientům umožňuje přístup ke službě pomocí vazby SOAP přes HTTP.</span><span class="sxs-lookup"><span data-stu-id="36c6b-118">The ~/service.svc/soap endpoint that allows the clients to access the service using the SOAP over HTTP binding.</span></span>  
  
 <span data-ttu-id="36c6b-119">Koncový bod HTTP je nakonfigurovaný pomocí <`webHttp`> standardní koncový bod, který má `helpEnabled` nastavené na `true`.</span><span class="sxs-lookup"><span data-stu-id="36c6b-119">The HTTP endpoint is configured with a <`webHttp`> standard endpoint which has `helpEnabled` set to `true`.</span></span> <span data-ttu-id="36c6b-120">Výsledkem je, že služba zpřístupňuje stránku s podporou standardu XHTML na adrese ~/Service.svc/http/Help, kterou mohou klienti využívající protokol HTTP použít pro přístup ke službě.</span><span class="sxs-lookup"><span data-stu-id="36c6b-120">As a result, the service exposes an XHTML based help page at ~/service.svc/http/help that HTTP-based clients can use to access the service.</span></span>  
  
 <span data-ttu-id="36c6b-121">Klientský projekt ukazuje přístup ke službě pomocí proxy protokolu SOAP (vygenerovaného prostřednictvím **Přidat odkaz na službu**) a přístupu ke službě pomocí <xref:System.Net.WebClient>.</span><span class="sxs-lookup"><span data-stu-id="36c6b-121">The client project demonstrates accessing the service using a SOAP proxy (generated through **Add Service Reference**) and accessing the service using <xref:System.Net.WebClient>.</span></span>  
  
 <span data-ttu-id="36c6b-122">Ukázka se skládá z webové služby a konzolové aplikace.</span><span class="sxs-lookup"><span data-stu-id="36c6b-122">The sample consists of a Web-hosted service and a console application.</span></span> <span data-ttu-id="36c6b-123">Při spuštění konzolové aplikace klient provede požadavky na službu a zapíše příslušné informace z odpovědí do okna konzoly.</span><span class="sxs-lookup"><span data-stu-id="36c6b-123">As the console application runs, the client makes requests to the service and writes the pertinent information from the responses to the console window.</span></span>
