# <a name="using-a-custom-expression-editor"></a>Použití editoru vlastních výrazů
Editor vlastních výrazů lze implementovat tak, aby poskytoval bohatší nebo jednodušší prostředí pro úpravy výrazů. Existuje několik scénářů, ve kterých můžete chtít použít editor vlastních výrazů:  
  
- Poskytování podpory pro technologie IntelliSense a další funkce pro bohaté úpravy v návrháři pracovních postupů s rehosted. Tato funkce musí být k dispozici, protože výchozí editor výrazů sady Visual Studio nelze použít v rehosted aplikacích.  
  
- Chcete-li zjednodušit prostředí pro úpravy výrazů pro uživatele obchodníanalytik, tak, aby například nutné se učit visual basic nebo zabývat se výrazy jazyka Visual Basic.  
  
 K implementaci editoru vlastních výrazů jsou zapotřebí tři základní kroky:  
  
1. Implementujte rozhraní <xref:System.Activities.Presentation.View.IExpressionEditorService>. Toto rozhraní spravuje vytváření a ničení editorů výrazů.  
  
2. Implementujte rozhraní <xref:System.Activities.Presentation.View.IExpressionEditorInstance>. Toto rozhraní implementuje uživatelské rozhraní pro úpravu výrazu uživatelského rozhraní.  
  
3. Publikujte <xref:System.Activities.Presentation.View.IExpressionEditorService> v aplikaci rehosted workflow.  
  
## <a name="implementing-a-custom-expression-editor-in-a-class-library"></a>Implementace editoru vlastních výrazů v knihovně tříd  
 Zde je ukázka kódu pro třídu `MyEditorService` (proof of <xref:System.Activities.Presentation.View.IExpressionEditorService> concept), která implementuje rozhraní je obsažena v projektu knihovny MyExpressionEditorService.  
  
```csharp  
using System;  
using System.Collections.Generic;  
using System.Activities.Presentation.View;  
using System.Activities.Presentation.Hosting;  
using System.Activities.Presentation.Model;  
  
namespace MyExpressionEditorService  
{  
    public class MyEditorService : IExpressionEditorService  
    {  
        public void CloseExpressionEditors()  
        {  
  
        }  
        public IExpressionEditorInstance CreateExpressionEditor(AssemblyContextControlItem assemblies, ImportedNamespaceContextItem importedNamespaces, List<ModelItem> variables, string text)  
        {  
            MyExpressionEditorInstance instance = new MyExpressionEditorInstance();  
            return instance;  
        }  
        public IExpressionEditorInstance CreateExpressionEditor(AssemblyContextControlItem assemblies, ImportedNamespaceContextItem importedNamespaces, List<ModelItem> variables, string text, System.Windows.Size initialSize)  
                {  
            MyExpressionEditorInstance instance = new MyExpressionEditorInstance();  
            return instance;  
        }  
        public IExpressionEditorInstance CreateExpressionEditor(AssemblyContextControlItem assemblies, ImportedNamespaceContextItem importedNamespaces, List<ModelItem> variables, string text, Type expressionType)  
            {  
            MyExpressionEditorInstance instance = new MyExpressionEditorInstance();  
            return instance;  
        }  
        public IExpressionEditorInstance CreateExpressionEditor(AssemblyContextControlItem assemblies, ImportedNamespaceContextItem importedNamespaces, List<ModelItem> variables, string text, Type expressionType, System.Windows.Size initialSize)  
        {  
            MyExpressionEditorInstance instance = new MyExpressionEditorInstance();  
            return instance;  
        }  
        public void UpdateContext(AssemblyContextControlItem assemblies, ImportedNamespaceContextItem importedNamespaces)  
        {  
  
        }  
  
    }  
}  
```

### <a name="publishing-a-custom-expression-editor-in-a-wpf-project"></a>Publikování editoru vlastních výrazů v projektu WPF  
 Zde je kód, který ukazuje, jak rehost návrháře v aplikaci `MyEditorService` WPF a jak vytvořit a publikovat službu. Před použitím tohoto kódu přidejte odkaz na projekt knihovny MyExpressionEditorService z projektu, který obsahuje aplikaci avalon2.  
  
```csharp  
using System.Windows;  
using System.Windows.Controls;  
using System.Activities.Presentation;  
using System.Activities.Statements;  
using System.Activities.Core.Presentation;  
using System.Activities.Presentation.View;  
using MyExpressionEditorService;  
  
namespace WpfApplication1  
{  
    /// <summary>  
    /// Interaction logic for MainWindow.xaml  
    /// </summary>  
    public partial class MainWindow : Window  
    {  
  
        private MyEditorService expressionEditorService;  
        public MainWindow()  
        {  
            InitializeComponent();  
            new DesignerMetadata().Register();  
            createDesigner();  
        }  
  
        public void createDesigner()  
        {  
            WorkflowDesigner designer = new WorkflowDesigner();  
            Sequence root = new Sequence()  
            {  
                Activities = {  
                new Assign(),  
                new WriteLine()}  
            };  
  
            designer.Load(root);  
  
            Grid.SetColumn(designer.View, 0);  
  
            // Create ExpressionEditorService
            this.expressionEditorService = new MyEditorService();  
  
            // Publish the instance of MyEditorService.  
            designer.Context.Services.Publish<IExpressionEditorService>(this.expressionEditorService);  
  
            MyGrid.Children.Add(designer.View);  
        }  
    }  
}  
```  
  
### <a name="notes"></a>Poznámky  
 Pokud používáte **expressiontextbox** ovládací prvek ve vlastní návrháře aktivit, není nutné vytvářet <xref:System.Activities.Presentation.View.IExpressionEditorService.CloseExpressionEditors%2A> a <xref:System.Activities.Presentation.View.IExpressionEditorService> ničit editory výrazů pomocí <xref:System.Activities.Presentation.View.IExpressionEditorService.CreateExpressionEditor%2A> a metody rozhraní. Třída <xref:System.Activities.Presentation.View.ExpressionTextBox> to zvládá za vás.
