## <a name="example"></a><span data-ttu-id="79958-141">Příklad</span><span class="sxs-lookup"><span data-stu-id="79958-141">Example</span></span>  
 <span data-ttu-id="79958-142">Následující příklad ukazuje způsob použití prvku `<trace>` k přidání `MyListener` naslouchacího procesu do kolekce `Listeners`.</span><span class="sxs-lookup"><span data-stu-id="79958-142">The following example shows how to use the `<trace>` element to add the listener `MyListener` to the `Listeners` collection.</span></span> <span data-ttu-id="79958-143">`MyListener` vytvoří soubor s názvem `MyListener.log` a zapíše výstup do souboru.</span><span class="sxs-lookup"><span data-stu-id="79958-143">`MyListener` creates a file that is named `MyListener.log` and writes the output to the file.</span></span> <span data-ttu-id="79958-144">Atribut `useGlobalLock` je nastaven na hodnotu `false`, což způsobí, že globální zámek nebude použit, pokud je naslouchací proces trasování bezpečný pro přístup z více vláken.</span><span class="sxs-lookup"><span data-stu-id="79958-144">The `useGlobalLock` attribute is set to `false`, which causes the global lock not to be used if the trace listener is thread safe.</span></span> <span data-ttu-id="79958-145">Atribut `autoflush` je nastaven na hodnotu `true`, což způsobí, že naslouchací proces trasování zapisuje do souboru bez ohledu na to, zda je volána metoda <xref:System.Diagnostics.Trace.Flush%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="79958-145">The `autoflush` attribute is set to `true`, which causes the trace listener to write to the file regardless of whether the <xref:System.Diagnostics.Trace.Flush%2A?displayProperty=nameWithType> method is called.</span></span> <span data-ttu-id="79958-146">Atribut `indentsize` je nastaven na hodnotu 0 (nula), což způsobí, že naslouchací proces odsadí nula mezer při volání metody <xref:System.Diagnostics.Trace.Indent%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="79958-146">The `indentsize` attribute is set to 0 (zero), which causes the listener to indent zero spaces when the <xref:System.Diagnostics.Trace.Indent%2A?displayProperty=nameWithType> method is called.</span></span>  
  
```xml  
<configuration>  
   <system.diagnostics>  
      <trace useGlobalLock="false" autoflush="true" indentsize="0">  
         <listeners>  
            <add name="myListener" type="System.Diagnostics.TextWriterTraceListener, system version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" initializeData="c:\myListener.log" />  
         </listeners>  
      </trace>  
   </system.diagnostics>  
</configuration>  
```
