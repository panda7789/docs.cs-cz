# <a name="compiler-error-cs0165"></a>Chyba kompilátoru CS0165

Použila se Nepřiřazená lokální proměnná Name.  
  
C# Kompilátor nepovoluje použití neinicializovaných proměnných. Pokud kompilátor zjistí použití proměnné, která pravděpodobně nebyla inicializována, vygeneruje chybu kompilátoru CS0165. Další informace najdete v tématu [pole](../../programming-guide/classes-and-structs/fields.md). Tato chyba je generována, když kompilátor narazí na konstrukce, která by mohla vést k použití nepřiřazené proměnné, a to i v případě, že váš konkrétní kód není. Předejdete tak nutnosti složitě složitá pravidla pro jednoznačné přiřazení.  
  
Další informace naleznete v tématu [Proč rekurzivní lambda nezpůsobí určitou chybu přiřazení?](https://docs.microsoft.com/archive/blogs/ericlippert/why-does-a-recursive-lambda-cause-a-definite-assignment-error).  
  
## <a name="example"></a>Příklad  

Následující ukázka generuje CS0165:  
  
```csharp  
// CS0165.cs  
using System;  
  
class MyClass  
{  
    public int i;  
}  
  
class MyClass2  
{  
    public static void Main(string[] args)  
    {  
        // i and j are not initialized.  
        int i, j;  
  
        // You can provide a value for args[0] in the 'Command line arguments'  
        // text box on the Debug tab of the project Properties window.  
        if (args[0] == "test")  
        {  
            i = 0;  
        }  
        // If the following else clause is absent, i might not be  
        // initialized.  
        //else  
        //{  
        //    i = 1;  
        //}  
  
        // Because i might not have been initialized, the following   
        // line causes CS0165.  
        j = i;  
  
        // To resolve the error, uncomment the else clause of the previous  
        // if statement, or initialize i when you declare it.  
  
        // The following example causes CS0165 because myInstance is  
        // declared but not instantiated.  
        MyClass myInstance;  
        // The following line causes the error.  
        myInstance.i = 0;   
  
        // To resolve the error, replace the previous declaration with  
        // the following line.  
        //MyClass myInstance = new MyClass();  
    }  
}  
```  
  
## <a name="example"></a>Příklad  

V definicích rekurzivního delegáta se může vyskytovat Chyba kompilátoru CS0165. Této chybě se můžete vyhnout tak, že definujete delegáta ve dvou příkazech, aby se proměnná před inicializací nepoužila. Následující příklad ukazuje chybu a řešení.  
  
```csharp  
class Program  
{  
    delegate void Del();  
    static void Main(string[] args)  
    {  
        // The following line causes CS0165 because variable d is used   
        // as an argument before it has been initialized.  
        Del d = delegate() { System.Console.WriteLine(d); };   
  
        //// To resolve the error, initialize d in a separate statement.  
        //Del d = null;  
        //// After d is initialized, you can use it as an argument.  
        //d = delegate() { System.Console.WriteLine(d); };  
        //d();  
    }  
}  
```
