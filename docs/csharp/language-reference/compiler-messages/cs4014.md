# <a name="compiler-warning-level-1-cs4014"></a><span data-ttu-id="6bb4e-102">Upozornění kompilátoru (úroveň 1) CS4014</span><span class="sxs-lookup"><span data-stu-id="6bb4e-102">Compiler Warning (level 1) CS4014</span></span>

<span data-ttu-id="6bb4e-103">Vzhledem k tomu, že toto volání není očekáváno, provádění aktuální metody pokračuje před dokončením volání.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-103">Because this call is not awaited, execution of the current method continues before the call is completed.</span></span> <span data-ttu-id="6bb4e-104">Zvažte použití operátoru "await" na výsledek volání.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-104">Consider applying the 'await' operator to the result of the call.</span></span>  
  
<span data-ttu-id="6bb4e-105">Aktuální metoda volá asynchronní metodu, která vrací <xref:System.Threading.Tasks.Task> a nebo <xref:System.Threading.Tasks.Task%601> a nepoužije operátor [await](../operators/await.md) na výsledek.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-105">The current method calls an async method that returns a <xref:System.Threading.Tasks.Task> or a <xref:System.Threading.Tasks.Task%601> and doesn’t apply the [await](../operators/await.md) operator to the result.</span></span> <span data-ttu-id="6bb4e-106">Volání asynchronní metody spustí asynchronní úlohu.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-106">The call to the async method starts an asynchronous task.</span></span> <span data-ttu-id="6bb4e-107">Protože však `await` není použit žádný operátor, program pokračuje bez čekání na dokončení úlohy.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-107">However, because no `await` operator is applied, the program continues without waiting for the task to complete.</span></span> <span data-ttu-id="6bb4e-108">Ve většině případů toto chování není to, co očekáváte.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-108">In most cases, that behavior isn't what you expect.</span></span> <span data-ttu-id="6bb4e-109">Obvykle další aspekty volající metody závisí na výsledcích volání nebo, minimálně, volaná metoda se očekává, že dokončení před návratem z metody, která obsahuje volání.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-109">Usually other aspects of the calling method depend on the results of the call or, minimally, the called method is expected to complete before you return from the method that contains the call.</span></span>  
  
<span data-ttu-id="6bb4e-110">Neméně důležitou otázkou je, co se stane s výjimkami, které jsou vyvolány v volané asynchronní metodě.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-110">An equally important issue is what happens to exceptions that are raised in the called async method.</span></span> <span data-ttu-id="6bb4e-111">Výjimka, která je vyvolána v <xref:System.Threading.Tasks.Task> <xref:System.Threading.Tasks.Task%601> metodě, která vrací nebo je uložena v vrácené úloze.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-111">An exception that's raised in a method that returns a <xref:System.Threading.Tasks.Task> or  <xref:System.Threading.Tasks.Task%601> is stored in the returned task.</span></span> <span data-ttu-id="6bb4e-112">Pokud nečekáte na úkol nebo explicitně zkontrolujte výjimky, výjimka je ztracena.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-112">If you don't await the task or explicitly check for exceptions, the exception is lost.</span></span> <span data-ttu-id="6bb4e-113">Pokud čekáte na úkol, jeho výjimka je znovu vyvolána.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-113">If you await the task, its exception is rethrown.</span></span>  

<span data-ttu-id="6bb4e-114">Jako osvědčený postup byste měli vždy počkat na hovor.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-114">As a best practice, you should always await the call.</span></span>

<span data-ttu-id="6bb4e-115">Měli byste zvážit potlačení upozornění pouze v případě, že jste si jisti, že nechcete čekat na dokončení asynchronního volání a že volaná metoda nevyvolá žádné výjimky.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-115">You should consider suppressing the warning only if you're sure that you don't want to wait for the asynchronous call to complete and that the called method won't raise any exceptions.</span></span> <span data-ttu-id="6bb4e-116">V takovém případě můžete potlačit upozornění přiřazením výsledku úkolu volání proměnné.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-116">In that case, you can suppress the warning by assigning the task result of the call to a variable.</span></span>  

<span data-ttu-id="6bb4e-117">Následující příklad ukazuje, jak způsobit upozornění, jak ho potlačit a jak čekat na volání.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-117">The following example shows how to cause the warning, how to suppress it, and how to await the call.</span></span>  

```csharp
async Task CallingMethodAsync()  
{  
    resultsTextBox.Text += "\r\n  Entering calling method.";  
    // Variable delay is used to slow down the called method so that you can  
    // distinguish between awaiting and not awaiting in the program's output.  
    // You can adjust the value to produce the output that this topic shows  
    // after the code.  
    var delay = 5000;  
  
    // Call #1.  
    // Call an async method. Because you don't await it, its completion
    // isn't coordinated with the current method, CallingMethodAsync.  
    // The following line causes warning CS4014.  
    CalledMethodAsync(delay);  
  
    // Call #2.  
    // To suppress the warning without awaiting, you can assign the
    // returned task to a variable. The assignment doesn't change how  
    // the program runs. However, recommended practice is always to  
    // await a call to an async method.  
  
    // Replace Call #1 with the following line.  
    //Task delayTask = CalledMethodAsync(delay);  
  
    // Call #3  
    // To contrast with an awaited call, replace the unawaited call
    // (Call #1 or Call #2) with the following awaited call. Best
    // practice is to await the call.  
  
    //await CalledMethodAsync(delay);  
  
    // If the call to CalledMethodAsync isn't awaited, CallingMethodAsync  
    // continues to run and, in this example, finishes its work and returns  
    // to its caller.  
    resultsTextBox.Text += "\r\n  Returning from calling method.";  
}  
  
async Task CalledMethodAsync(int howLong)  
{  
    resultsTextBox.Text +=
        "\r\n    Entering called method, starting and awaiting Task.Delay.";  
  
    // Slow the process down a little so that you can distinguish between  
    // awaiting and not awaiting in the program's output. Adjust the value  
    // for howLong if necessary.  
    await Task.Delay(howLong);  
    resultsTextBox.Text +=
        "\r\n    Task.Delay is finished--returning from called method.";  
}  
```

<span data-ttu-id="6bb4e-118">V příkladu pokud zvolíte Volání #1 nebo Volání #2,`CalledMethodAsync`neočekávaná asynchronní`CallingMethodAsync`metoda ( ) skončí`startButton_Click`po dokončení volajícího ( ) i volajícího ( ).</span><span class="sxs-lookup"><span data-stu-id="6bb4e-118">In the example, if you choose Call #1 or Call #2, the unawaited async method (`CalledMethodAsync`) finishes after both its caller (`CallingMethodAsync`) and the caller's caller (`startButton_Click`) are complete.</span></span> <span data-ttu-id="6bb4e-119">Poslední řádek v následujícím výstupu ukazuje, když volaná metoda skončí.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-119">The last line in the following output shows you when the called method finishes.</span></span> <span data-ttu-id="6bb4e-120">Vstup do a ukončení z `CallingMethodAsync` obslužné rutiny události, která volá v plném příkladu jsou označeny ve výstupu.</span><span class="sxs-lookup"><span data-stu-id="6bb4e-120">Entry to and exit from the event handler that calls `CallingMethodAsync` in the full example are marked in the output.</span></span>
