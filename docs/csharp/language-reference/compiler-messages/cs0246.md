# <a name="compiler-error-cs0246"></a>Chyba kompilátoru CS0246
Typ nebo název oboru názvů ' Type/namespace ' nebyl nalezen (nechybí Direktiva using nebo odkaz na sestavení?)  
  
 Typ nebo obor názvů, který se používá v programu, nebyl nalezen. Možná jste zapomněli odkazovat na sestavení, které obsahuje daný typ ([-reference](../compiler-options/reference-compiler-option.md)), nebo pokud jste nepřidali požadovanou [direktivu using](../keywords/using-directive.md).  Nebo se může jednat o problém se sestavením, na které se pokoušíte odkazovat.  
  
 V následujících situacích dojde k chybě kompilátoru CS0246.  
  
- Nezapomněli jste zadat název typu nebo oboru názvů? Bez správného názvu kompilátor nemůže najít definici pro typ nebo obor názvů. K tomu často dochází, protože použití malých a velkých písmen v názvu typu není správné. Například `Dataset ds;` generuje CS0246, protože musí být v datové sadě velká písmena.  
  
- Pokud je chyba pro název oboru názvů, Přidali jste odkaz ([-reference](../compiler-options/reference-compiler-option.md)) na sestavení, které obsahuje obor názvů? Kód může například obsahovat direktivu `using Accessibility`. Pokud však projekt neodkazuje na přístupnost sestavení. dll, je hlášena chyba CS0246. Další informace naleznete v tématu [Správa odkazů v projektu](/visualstudio/ide/managing-references-in-a-project) .  
  
- Pokud se jedná o chybu pro název typu, zahrnuli jste správnou [direktivu using](../keywords/using-directive.md), nebo případně plně kvalifikováni název typu? Zvažte následující deklaraci: `DataSet ds`. Pokud chcete použít `DataSet` typ, potřebujete dvě věci. Nejprve potřebujete odkaz na sestavení, které obsahuje definici pro typ `DataSet`. Za druhé budete potřebovat direktivu `using` pro obor názvů, kde se nachází `DataSet`. Například vzhledem k tomu, že `DataSet` je umístěn v oboru názvů **System. data** , budete potřebovat následující direktivu na začátku kódu: `using System.Data`.  
  
     Direktiva `using` není povinná. Pokud však tuto direktivu vynecháte, je při odkazování na ni nutné plně kvalifikovat typ `DataSet`. Úplná kvalifikace znamená, že zadáte obor názvů i typ pokaždé, když budete odkazovat na typ v kódu. Vynecháte-li direktivu `using` v předchozím příkladu, je nutné napsat `System.Data.DataSet ds` pro deklaraci `ds` namísto `DataSet ds`.  
  
- Použili jste proměnnou nebo jiný prvek jazyka, kde byl očekáván typ? Například v příkazu **is** , pokud místo skutečného typu použijete `Type` objekt, zobrazí se chyba CS0246.  

- Odkazovali jste na sestavení, které bylo vytvořeno proti vyšší verzi architektury než cílová architektura programu? Nebo jste odkazovali na projekt, který cílí na vyšší verzi architektury než cílová architektura programu? Například pracujete na projektu, který cílí na .NET Framework 4.6.1, a použijete typ z projektu, který cílí na .NET Framework 4.7.1. Pak se zobrazí chyba CS0246.
  
- Použili jste *direktivu using alias* bez úplného kvalifikovaného názvu typu? Direktiva `using` alias nepoužívá direktivy `using` v souboru zdrojového kódu k překladu typů. Následující příklad generuje CS0246, protože typ `List<int>` není plně kvalifikovaný. Direktiva `using` pro `System.Collections.Generic` nebrání chybě.  
  
    ```csharp  
    using System.Collections.Generic;  
  
    // The following declaration generates CS0246.  
    using myAliasName = List<int>;   
  
    // To avoid the error, fully qualify List.  
    using myAliasName2 = System.Collections.Generic.List<int>;  
    ```  
  
     Pokud se zobrazí tato chyba v kódu, který dříve pracoval, vyhledá se v Průzkumník řešení chybějící nebo nevyřešené odkazy. Potřebujete znovu nainstalovat balíček [NuGet](https://www.nuget.org/) ? Informace o tom, jak systém sestavení vyhledává odkazy, naleznete v tématu [řešení odkazů na soubory v týmovém sestavení](https://docs.microsoft.com/archive/blogs/manishagarwal/resolving-file-references-in-team-build-part-2). Pokud se zdá, že jsou všechny odkazy správné, vyhledejte v historii správy zdrojového kódu, co se změnilo v souboru. csproj nebo v místním zdrojovém souboru.  
  
     Pokud jste ještě nepoužili odkaz, použijte Prohlížeč objektů pro zkontrolování sestavení, které by mělo obsahovat tento obor názvů, a ověřte, zda je obor názvů přítomen. Pokud ověříte Prohlížeč objektů, že sestavení obsahuje obor názvů, zkuste odebrat direktivu `using` pro obor názvů a podívejte se, co ostatní přeruší. Kořenový problém může být s jiným typem v jiném sestavení.  
  
 Následující příklad generuje CS0246, protože chybí nezbytná direktiva `using`.  
  
```csharp  
// CS0246.cs  
//using System.Diagnostics;  
  
public class MyClass  
{  
    // The following line causes CS0246. To fix the error, uncomment  
    // the using directive for the namespace for this attribute,  
    // System.Diagnostics.  
    [Conditional("A")]  
    public void Test()  
    {  
    }  
  
    public static void Main()  
    {  
    }  
}  
```  
  
 Následující příklad způsobí CS0246, protože byl použit objekt typu `Type`, kde byl očekáván skutečný typ.  
  
```csharp  
// CS0246b.cs  
using System;  
  
class ExampleClass  
{  
    public bool supports(object o, Type t)  
    {  
        // The following line causes CS0246. You must use an  
        // actual type, such as ExampleClass, String, or Type.  
        if (o is t)  
        {  
            return true;  
        }  
        return false;  
    }  
}  
  
class Program  
{  
    public static void Main()  
    {  
        ExampleClass myC = new ExampleClass();  
        myC.supports(myC, myC.GetType());  
    }  
}  
```
