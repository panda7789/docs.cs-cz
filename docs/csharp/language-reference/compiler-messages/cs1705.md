## <a name="example"></a><span data-ttu-id="dc0b8-119">Příklad</span><span class="sxs-lookup"><span data-stu-id="dc0b8-119">Example</span></span>  
 <span data-ttu-id="dc0b8-120">Následující kód odkazuje na dvě verze dll, které jsou definovány v předchozím kódu.</span><span class="sxs-lookup"><span data-stu-id="dc0b8-120">The following code references the two DLL versions that are defined in the preceding code.</span></span> <span data-ttu-id="dc0b8-121">`AssemblyA`odkazuje na dll vytvořenou CS1705a.cs (verze 1.0).</span><span class="sxs-lookup"><span data-stu-id="dc0b8-121">`AssemblyA` refers to the DLL created by CS1705a.cs (version 1.0).</span></span> <span data-ttu-id="dc0b8-122">`AssemblyB`odkazuje na dll vytvořenou CS1705b.cs (verze 2.0).</span><span class="sxs-lookup"><span data-stu-id="dc0b8-122">`AssemblyB` refers to the DLL created by CS1705b.cs (version 2.0).</span></span> <span data-ttu-id="dc0b8-123">V `ClassC`, jsou definovány dvě metody.</span><span class="sxs-lookup"><span data-stu-id="dc0b8-123">In `ClassC`, two methods are defined.</span></span> <span data-ttu-id="dc0b8-124">První , `Return1A`vrátí objekt typu `Class1A`, což je `Class1` alias z verze 1.0 knihovny DLL.</span><span class="sxs-lookup"><span data-stu-id="dc0b8-124">The first, `Return1A`, returns an object of type `Class1A`, which is an alias for `Class1` from version 1.0 of the DLL.</span></span> <span data-ttu-id="dc0b8-125">Druhý , `Return1B`vrátí objekt typu `Class1B`, což je `Class1` alias z verze 2.0 knihovny DLL.</span><span class="sxs-lookup"><span data-stu-id="dc0b8-125">The second, `Return1B`, returns an object of type `Class1B`, which is an alias for `Class1` from version 2.0 of the DLL.</span></span> <span data-ttu-id="dc0b8-126">Definice vytvoří `Return1A` závislost na verzi 1.0; definice vytvoří `Return1B` závislost na verzi 2.0.</span><span class="sxs-lookup"><span data-stu-id="dc0b8-126">The definition of `Return1A` creates a dependency on version 1.0; the definition of `Return1B` creates a dependency on version 2.0.</span></span>  
  
```csharp  
// CS1705c.cs  
  
// Compile by using the following command. AssemblyA refers to the DLL created by  
// CS1705a.cs (version 1.0). AssemblyB refers to the DLL created by CS1705b.cs  
// (version 2.0).  
//    csc /target:library /r:AssemblyA=C:\\CS1705.dll /r:AssemblyB=CS1705.dll CS1705c.cs  
  
extern alias AssemblyA;  
extern alias AssemblyB;  
  
// Class1A is an alias for type Class1 from VS1705a.cs, which is in version 1.0
// of the assembly. Class1B is an alias for type Class1 from CS1705b.cs, which  
// is in version 2.0 of the assembly.  
  
using Class1A = AssemblyA::Class1;  
using Class1B = AssemblyB::Class1;  
  
// Method Return1A in ClassC returns an object of type Class1A, which is  
// Class1 from version 1.0 of the DLL. Method Return1B returns an object  
// of type Class1B, which is Class1 from version 2.0 of the DLL.  
  
public class ClassC  
{  
    // The following line creates a dependency on version 1.0 of CS1705.dll.  
    // This is not the source of the problem when ClassC is accessed from  
    // CS1705d.cs because CS1705d.cs references version 1.0 of the DLL.
    // Therefore, type Class1A and the assembly have the same version.  
    public static Class1A Return1A() { return new Class1A(); }  
  
    // The following line creates a dependency on version 2.0 of CS1705.dll.  
    // This causes compiler error CS1705 when ClassC is accessed from
    // CS1705d.cs, because CS1705d.cs does not reference version 2.0 of the
    // DLL. Class1B is the alias for Class1 in version 2.0, and CS1705d.cs
    // references version 1.0.  
    public static Class1B Return1B() { return new Class1B(); }  
}  
```
