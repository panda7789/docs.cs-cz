# <a name="compiler-error-cs1612"></a><span data-ttu-id="fe7f3-102">Chyba kompilátoru CS1612</span><span class="sxs-lookup"><span data-stu-id="fe7f3-102">Compiler Error CS1612</span></span>
<span data-ttu-id="fe7f3-103">Zpětnou hodnotu výrazu nelze změnit, protože se nejedná o proměnnou.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-103">Cannot modify the return value of 'expression' because it is not a variable</span></span>  
  
 <span data-ttu-id="fe7f3-104">Došlo k pokusu o změnu typu hodnoty, který je vytvořen jako výsledek zprostředkujícího výrazu, ale není uložen v proměnné.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-104">An attempt was made to modify a value type that is produced as the result of an intermediate expression but is not stored in a variable.</span></span> <span data-ttu-id="fe7f3-105">K této chybě může dojít, když se pokusíte přímo upravit strukturu v obecné kolekci, jak je znázorněno v následujícím příkladu:</span><span class="sxs-lookup"><span data-stu-id="fe7f3-105">This error can occur when you attempt to directly modify a struct in a generic collection, as shown in the following example:</span></span>  
  
```csharp  
List<myStruct> list = {…};  
list[0].Name = "MyStruct42"; //CS1612  
```  
  
 <span data-ttu-id="fe7f3-106">Chcete-li upravit strukturu, nejprve ji přiřaďte k místní proměnné, upravte proměnnou a pak přiřaďte proměnnou zpět k položce v kolekci.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-106">To modify the struct, first assign it to a local variable, modify the variable, then assign the variable back to the item in the collection.</span></span>  
  
```csharp  
List<myStruct> list = {…};  
MyStruct ms = list[0];  
ms.Name = "MyStruct42";  
list[0] = ms;  
```  
  
 <span data-ttu-id="fe7f3-107">K této chybě dochází, protože typy hodnot jsou zkopírovány při přiřazení.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-107">This error occurs because value types are copied on assignment.</span></span> <span data-ttu-id="fe7f3-108">Při načtení typu hodnoty z vlastnosti nebo indexeru, získáváte kopii objektu, nikoli odkaz na samotný objekt.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-108">When you retrieve a value type from a property or indexer, you are getting a copy of the object, not a reference to the object itself.</span></span> <span data-ttu-id="fe7f3-109">Kopie, která je vrácena není uloženvlastnost nebo indexer, protože jsou ve skutečnosti metody, nikoli umístění úložiště (proměnné).</span><span class="sxs-lookup"><span data-stu-id="fe7f3-109">The copy that is returned is not stored by the property or indexer because they are actually methods, not storage locations (variables).</span></span> <span data-ttu-id="fe7f3-110">Před úpravou je nutné uložit kopii do proměnné, kterou deklarujete.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-110">You must store the copy into a variable that you declare before you can modify it.</span></span>  
  
 <span data-ttu-id="fe7f3-111">K chybě nedochází u typů odkazů, protože vlastnost nebo indexer v takovém případě vrátí odkaz na existující objekt, což je umístění úložiště.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-111">The error does not occur with reference types because a property or indexer in that case returns a reference to an existing object, which is a storage location.</span></span>  
  
 <span data-ttu-id="fe7f3-112">Pokud definujete třídu nebo strukturu, můžete tuto chybu vyřešit úpravou deklarace vlastnosti tak, aby poskytovala přístup členům struktury.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-112">If you are defining the class or struct, you can resolve this error by modifying your property declaration to provide access to the members of a struct.</span></span> <span data-ttu-id="fe7f3-113">Pokud píšete kód klienta, můžete chybu vyřešit vytvořením vlastní instance struktury, úpravou jejích polí a přiřazením celé struktury zpět do vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-113">If you are writing client code, you can resolve the error by creating your own instance of the struct, modifying its fields, and then assigning the entire struct back to the property.</span></span> <span data-ttu-id="fe7f3-114">Jako třetí alternativu můžete změnit strukturu na třídu.</span><span class="sxs-lookup"><span data-stu-id="fe7f3-114">As a third alternative, you can change your struct to a class.</span></span>  
  
## <a name="example"></a><span data-ttu-id="fe7f3-115">Příklad</span><span class="sxs-lookup"><span data-stu-id="fe7f3-115">Example</span></span>  
 <span data-ttu-id="fe7f3-116">CS1612 také dochází při pokusu o přístup k člen struktury prostřednictvím vlastnosti na ohraničující třídy, která vrací celou strukturu, jak je znázorněno v následujícím příkladu:</span><span class="sxs-lookup"><span data-stu-id="fe7f3-116">CS1612 also occurs when you attempt to access the member of a struct through a property on an enclosing class that is returning the entire struct, as shown in the following example:</span></span>  
  
```csharp  
// CS1612.cs  
using System;  
  
public struct MyStruct  
{  
    public int Width;  
}  
  
public class ListView  
{  
    MyStruct ms;  
    public MyStruct Size  
    {  
        get { return ms; }  
        set { ms = value; }  
    }  
}  
  
public class MyClass  
{  
    public MyClass()  
    {  
        ListView lvi;  
        lvi = new ListView();  
        lvi.Size.Width = 5; // CS1612  
  
        // You can use the following lines instead.  
        // MyStruct ms;  
        // ms.Width = 5;  
        // lvi.Size = ms;
    }  
  
    public static void Main()
    {  
        MyClass mc = new MyClass();  
        // Keep the console open in debug mode.  
        Console.WriteLine("Press any key to exit.");  
        Console.ReadKey();
    }  
}  
```
