# <a name="compiler-error-cs0269"></a>Chyba kompilátoru CS0269
Použití nepřiřazeného out parametru 'parametr'  
  
 Kompilátor nemohl ověřit, že parametr out byl přiřazen hodnotu před jeho použitím; jeho hodnota může být při přiřazení nedefinovaná. Nezapomeňte přiřadit hodnotu `out` parametrům ve volané metodě před přístupem k hodnotě. Pokud potřebujete použít hodnotu proměnné předané, `ref` použijte parametr místo. Další informace naleznete v tématu [Předávání parametrů](../../programming-guide/classes-and-structs/passing-parameters.md).  
  
## <a name="example"></a>Příklad  
 Následující ukázka generuje CS0269:  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a>Příklad  
 K tomu může dojít také v případě, že dojde k inicializaci proměnné v bloku try, který kompilátor nemůže ověřit, bude úspěšně spuštěn:  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
