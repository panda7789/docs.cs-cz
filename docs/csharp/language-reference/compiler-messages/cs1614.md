# <a name="compiler-error-cs1614"></a><span data-ttu-id="dcbb4-103">Chyba kompilátoru CS1614</span><span class="sxs-lookup"><span data-stu-id="dcbb4-103">Compiler Error CS1614</span></span>
<span data-ttu-id="dcbb4-104">název Name je nejednoznačný mezi "Name" a "nameAttribute"; použijte buď @name ' ' nebo ' nameAttribute '.</span><span class="sxs-lookup"><span data-stu-id="dcbb4-104">'name' is ambiguous between 'name' and 'nameAttribute'; use either '@name' or 'nameAttribute'.</span></span>
  
 <span data-ttu-id="dcbb4-105">V kompilátoru se vyskytla nejednoznačná specifikace atributu.</span><span class="sxs-lookup"><span data-stu-id="dcbb4-105">The compiler has encountered an ambiguous attribute specification.</span></span>  
  
 <span data-ttu-id="dcbb4-106">Pro usnadnění práce kompilátor C# umožňuje zadat **ExampleAttribute** jako pouze `[Example]` .</span><span class="sxs-lookup"><span data-stu-id="dcbb4-106">For convenience, the C# compiler allows you to specify **ExampleAttribute** as just `[Example]`.</span></span> <span data-ttu-id="dcbb4-107">Nejednoznačnost však nastane, pokud třída atributu s názvem `Example` existuje společně s **ExampleAttribute**, protože kompilátor nemůže říct, zda `[Example]` odkazuje na `Example` atribut nebo atribut **ExampleAttribute** .</span><span class="sxs-lookup"><span data-stu-id="dcbb4-107">However, ambiguity arises if an attribute class named `Example` exists along with **ExampleAttribute**, because the compiler cannot tell if `[Example]` refers to the `Example` attribute or the **ExampleAttribute** attribute.</span></span> <span data-ttu-id="dcbb4-108">Pro objasnění použijte `[@Example]` pro `Example` atribut a `[ExampleAttribute]` pro **ExampleAttribute**.</span><span class="sxs-lookup"><span data-stu-id="dcbb4-108">To clarify, use `[@Example]` for the `Example` attribute and `[ExampleAttribute]` for **ExampleAttribute**.</span></span>  
  
 <span data-ttu-id="dcbb4-109">Následující ukázka generuje CS1614:</span><span class="sxs-lookup"><span data-stu-id="dcbb4-109">The following sample generates CS1614:</span></span>  
  
```csharp  
// CS1614.cs  
using System;  
  
// Both of the following classes are valid attributes with valid  
// names (MySpecial and MySpecialAttribute). However, because the lookup  
// rules for attributes involves auto-appending the 'Attribute' suffix  
// to the identifier, these two attributes become ambiguous; that is,  
// if you specify MySpecial, the compiler can't tell if you want  
// MySpecial or MySpecialAttribute.  
  
public class MySpecial : Attribute {  
   public MySpecial() {}  
}  
  
public class MySpecialAttribute : Attribute {  
   public MySpecialAttribute() {}  
}  
  
class MakeAWarning {  
   [MySpecial()] // CS1614  
                 // Ambiguous: MySpecial or MySpecialAttribute?  
   public static void Main() {  
   }  
  
   [@MySpecial()] // This isn't ambiguous, it binds to the first attribute above.  
   public static void NoWarning() {  
   }  
  
   [MySpecialAttribute()] // This isn't ambiguous, it binds to the second attribute above.  
   public static void NoWarning2() {  
   }  
  
   [@MySpecialAttribute()] // This is also legal.  
   public static void NoWarning3() {  
   }  
}  
```
