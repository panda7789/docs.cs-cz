<span data-ttu-id="629ae-104">Bitový operátor OR se používá pro operand s rozšířeným znaménkem; nejprve zvažte přetypování na menší typ bez znaménka.</span><span class="sxs-lookup"><span data-stu-id="629ae-104">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="629ae-105">Kompilátor implicitně rozšířil proměnnou a rozšířila znaménko a pak použil výslednou hodnotu v bitové operaci OR.</span><span class="sxs-lookup"><span data-stu-id="629ae-105">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="629ae-106">To může vést k neočekávanému chování.</span><span class="sxs-lookup"><span data-stu-id="629ae-106">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="629ae-107">Následující ukázka generuje CS0675:</span><span class="sxs-lookup"><span data-stu-id="629ae-107">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
