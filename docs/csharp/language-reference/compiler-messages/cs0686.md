## <a name="example"></a><span data-ttu-id="fe2ad-110">Příklad</span><span class="sxs-lookup"><span data-stu-id="fe2ad-110">Example</span></span>  
 <span data-ttu-id="fe2ad-111">Následující ukázka generuje CS0686:</span><span class="sxs-lookup"><span data-stu-id="fe2ad-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="fe2ad-112">Příklad</span><span class="sxs-lookup"><span data-stu-id="fe2ad-112">Example</span></span>  
 <span data-ttu-id="fe2ad-113">K této chybě může dojít také při deklarování událostí.</span><span class="sxs-lookup"><span data-stu-id="fe2ad-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="fe2ad-114">Konstrukce události automaticky generuje `add_event` `remove_event` metody a, které by mohly být v konfliktu s metodami se stejným názvem v rozhraní, jako v následující ukázce:</span><span class="sxs-lookup"><span data-stu-id="fe2ad-114">The event construct automatically generates the `add_event` and `remove_event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>

// CS0686b.cs

interface I  
{  
    void add_OnMyEvent(EventHandler e);
