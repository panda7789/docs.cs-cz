## <a name="deferred-execution"></a><span data-ttu-id="3ff14-107">Odložené provedení</span><span class="sxs-lookup"><span data-stu-id="3ff14-107">Deferred Execution</span></span>  
 <span data-ttu-id="3ff14-108">Odložené spuštění znamená, že vyhodnocení výrazu je zpožděno, dokud není skutečně vyžadováno jeho *realizovaná* hodnota.</span><span class="sxs-lookup"><span data-stu-id="3ff14-108">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="3ff14-109">Odložené provádění může významně zlepšit výkon, pokud je třeba manipulovat s velkými datovými kolekcemi, zejména v programech, které obsahují sérii zřetězených dotazů nebo manipulace.</span><span class="sxs-lookup"><span data-stu-id="3ff14-109">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="3ff14-110">V nejlepším případě se odložené spouštění povoluje pouze jedna iterace prostřednictvím zdrojové kolekce.</span><span class="sxs-lookup"><span data-stu-id="3ff14-110">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>  
  
 <span data-ttu-id="3ff14-111">Technologie LINQ využívají odložené spouštění jak v rámci členů základních <xref:System.Linq?displayProperty=nameWithType> tříd, tak v metodách rozšíření v různých oborech názvů LINQ, jako je například <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType> .</span><span class="sxs-lookup"><span data-stu-id="3ff14-111">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="3ff14-112">Odložené provádění je podporováno přímo v jazyce C# pomocí klíčového slova [yield](../../../language-reference/keywords/yield.md) (ve formě `yield-return` příkazu) při použití v rámci bloku iterátoru.</span><span class="sxs-lookup"><span data-stu-id="3ff14-112">Deferred execution is supported directly in the C# language by the [yield](../../../language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="3ff14-113">Takový iterátor musí vracet kolekci typu <xref:System.Collections.IEnumerator> nebo <xref:System.Collections.Generic.IEnumerator%601> (nebo odvozeného typu).</span><span class="sxs-lookup"><span data-stu-id="3ff14-113">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>  
  
## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="3ff14-114">Eager vs. opožděné hodnocení</span><span class="sxs-lookup"><span data-stu-id="3ff14-114">Eager vs. Lazy Evaluation</span></span>  
 <span data-ttu-id="3ff14-115">Když napíšete metodu, která implementuje odložené vykonání, je také nutné rozhodnout, zda implementovat metodu pomocí opožděného vyhodnocení nebo vyhodnocení Eager.</span><span class="sxs-lookup"><span data-stu-id="3ff14-115">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>  
  
- <span data-ttu-id="3ff14-116">V *opožděném vyhodnocení*je během každého volání iterátoru zpracován jediný element zdrojové kolekce.</span><span class="sxs-lookup"><span data-stu-id="3ff14-116">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="3ff14-117">Toto je typický způsob, jakým jsou implementovány iterátory.</span><span class="sxs-lookup"><span data-stu-id="3ff14-117">This is the typical way in which iterators are implemented.</span></span>  
  
- <span data-ttu-id="3ff14-118">Při *vyhodnocování Eager*bude první volání iterátoru mít za následek zpracování celé kolekce.</span><span class="sxs-lookup"><span data-stu-id="3ff14-118">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="3ff14-119">Může být také nutné zadat dočasnou kopii zdrojové kolekce.</span><span class="sxs-lookup"><span data-stu-id="3ff14-119">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="3ff14-120">Například <xref:System.Linq.Enumerable.OrderBy%2A> Metoda musí seřadit celou kolekci před tím, než vrátí první prvek.</span><span class="sxs-lookup"><span data-stu-id="3ff14-120">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>  
  
 <span data-ttu-id="3ff14-121">Opožděné hodnocení obvykle poskytuje lepší výkon, protože distribuuje režijní náklady rovnoměrně během hodnocení kolekce a minimalizuje používání dočasných dat.</span><span class="sxs-lookup"><span data-stu-id="3ff14-121">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="3ff14-122">Samozřejmě není pro některé operace žádná jiná možnost než vyhodnotit mezilehlé výsledky.</span><span class="sxs-lookup"><span data-stu-id="3ff14-122">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
