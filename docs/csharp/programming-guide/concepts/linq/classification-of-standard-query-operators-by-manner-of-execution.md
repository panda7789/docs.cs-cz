## <a name="manners-of-execution"></a><span data-ttu-id="d4990-108">Způsoby provedení</span><span class="sxs-lookup"><span data-stu-id="d4990-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="d4990-109">Okamžité</span><span class="sxs-lookup"><span data-stu-id="d4990-109">Immediate</span></span>  
 <span data-ttu-id="d4990-110">Okamžité spuštění znamená, že zdroj dat je číst a operace se provádí v bodě v kódu, kde je deklarován dotaz.</span><span class="sxs-lookup"><span data-stu-id="d4990-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="d4990-111">Všechny standardní operátory dotazu, které vrátí jeden, nevýčíslitelný výsledek spustit okamžitě.</span><span class="sxs-lookup"><span data-stu-id="d4990-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="d4990-112">Odložené</span><span class="sxs-lookup"><span data-stu-id="d4990-112">Deferred</span></span>  
 <span data-ttu-id="d4990-113">Odložené spuštění znamená, že operace není provedena v bodě v kódu, kde je deklarován dotaz.</span><span class="sxs-lookup"><span data-stu-id="d4990-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="d4990-114">Operace se provádí pouze v případě, že je proměnná `foreach` dotazu uvedena, například pomocí příkazu.</span><span class="sxs-lookup"><span data-stu-id="d4990-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="d4990-115">To znamená, že výsledky spuštění dotazu závisí na obsahu zdroje dat při spuštění dotazu, nikoli při definování dotazu.</span><span class="sxs-lookup"><span data-stu-id="d4990-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="d4990-116">Pokud je proměnná dotazu výčtu vícekrát, výsledky se mohou lišit pokaždé.</span><span class="sxs-lookup"><span data-stu-id="d4990-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="d4990-117">Téměř všechny standardní operátory dotazu, jejichž návratový typ je <xref:System.Collections.Generic.IEnumerable%601> nebo <xref:System.Linq.IOrderedEnumerable%601> spustit odložené způsobem.</span><span class="sxs-lookup"><span data-stu-id="d4990-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="d4990-118">Operátory dotazů, které používají odložené spuštění lze navíc klasifikovat jako streamování nebo jiné datové proudy.</span><span class="sxs-lookup"><span data-stu-id="d4990-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="d4990-119">Streamování</span><span class="sxs-lookup"><span data-stu-id="d4990-119">Streaming</span></span>  
 <span data-ttu-id="d4990-120">Operátory streamování nemusí číst všechna zdrojová data před tím, než poskytují prvky.</span><span class="sxs-lookup"><span data-stu-id="d4990-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="d4990-121">V době spuštění operátor streamování provádí svou operaci na každý zdrojový prvek při čtení a dává prvek v případě potřeby.</span><span class="sxs-lookup"><span data-stu-id="d4990-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="d4990-122">Operátor streamování pokračuje ve čtení zdrojových prvků, dokud nelze vytvořit prvek výsledku.</span><span class="sxs-lookup"><span data-stu-id="d4990-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="d4990-123">To znamená, že více než jeden zdrojový prvek může být přečten a vytvořit jeden prvek výsledku.</span><span class="sxs-lookup"><span data-stu-id="d4990-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="d4990-124">Nedatové přenosy</span><span class="sxs-lookup"><span data-stu-id="d4990-124">Non-Streaming</span></span>  
 <span data-ttu-id="d4990-125">Operátory bez streamování musí číst všechna zdrojová data, než mohou přinést prvek výsledku.</span><span class="sxs-lookup"><span data-stu-id="d4990-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="d4990-126">Operace, jako je řazení nebo seskupení spadají do této kategorie.</span><span class="sxs-lookup"><span data-stu-id="d4990-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="d4990-127">V době spuštění operátory dotazů bez streamování číst všechna zdrojová data, dát do datové struktury, provést operaci a výnos výsledné prvky.</span><span class="sxs-lookup"><span data-stu-id="d4990-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
