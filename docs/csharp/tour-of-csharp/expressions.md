<span data-ttu-id="2049c-108">Pokud výraz obsahuje více operátorů, *Priorita* operátorů řídí pořadí, ve kterém jsou jednotlivé operátory vyhodnocovány.</span><span class="sxs-lookup"><span data-stu-id="2049c-108">When an expression contains multiple operators, the *precedence* of the operators controls the order in which the individual operators are evaluated.</span></span> <span data-ttu-id="2049c-109">Například výraz `x + y * z` je vyhodnocen jako `x + (y * z)`, protože operátor `*` má vyšší prioritu než operátor `+`.</span><span class="sxs-lookup"><span data-stu-id="2049c-109">For example, the expression `x + y * z` is evaluated as `x + (y * z)` because the `*` operator has higher precedence than the `+` operator.</span></span>

<span data-ttu-id="2049c-110">Když dojde k operandu mezi dvěma operátory se stejnou prioritou, *asociativita* operátor řídí pořadí, ve kterém jsou operace prováděny:</span><span class="sxs-lookup"><span data-stu-id="2049c-110">When an operand occurs between two operators with the same precedence, the *associativity* of the operators controls the order in which the operations are performed:</span></span>

* <span data-ttu-id="2049c-111">S výjimkou operátorů přiřazení a slučování s hodnotou null jsou všechny binární operátory *asociativní*, což znamená, že operace jsou prováděny zleva doprava.</span><span class="sxs-lookup"><span data-stu-id="2049c-111">Except for the assignment and null-coalescing operators, all binary operators are *left-associative*, meaning that operations are performed from left to right.</span></span> <span data-ttu-id="2049c-112">Například `x + y + z` je vyhodnocen jako `(x + y) + z`.</span><span class="sxs-lookup"><span data-stu-id="2049c-112">For example, `x + y + z` is evaluated as `(x + y) + z`.</span></span>
* <span data-ttu-id="2049c-113">Operátory přiřazení, `??` a `??=` operátory pro sloučení s hodnotou null a podmíněný operátor `?:` jsou *asociativní zprava*, což znamená, že operace jsou prováděny zprava doleva.</span><span class="sxs-lookup"><span data-stu-id="2049c-113">The assignment operators, the null-coalescing `??` and `??=` operators, and the conditional operator `?:` are *right-associative*, meaning that operations are performed from right to left.</span></span> <span data-ttu-id="2049c-114">Například `x = y = z` je vyhodnocen jako `x = (y = z)`.</span><span class="sxs-lookup"><span data-stu-id="2049c-114">For example, `x = y = z` is evaluated as `x = (y = z)`.</span></span>

<span data-ttu-id="2049c-115">Priority a asociativita lze ovládat pomocí závorek.</span><span class="sxs-lookup"><span data-stu-id="2049c-115">Precedence and associativity can be controlled using parentheses.</span></span> <span data-ttu-id="2049c-116">Například `x + y * z` nejprve vynásobí `y` `z` a pak výsledek přidá do `x`, ale `(x + y) * z` nejprve přidá `x` a `y` a pak výsledek vynásobí `z`.</span><span class="sxs-lookup"><span data-stu-id="2049c-116">For example, `x + y * z` first multiplies `y` by `z` and then adds the result to `x`, but `(x + y) * z` first adds `x` and `y` and then multiplies the result by `z`.</span></span>
