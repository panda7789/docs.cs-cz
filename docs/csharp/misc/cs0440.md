# <a name="compiler-warning-level-2-cs0440"></a><span data-ttu-id="247a0-103">Upozornění kompilátoru (úroveň 2) CS0440</span><span class="sxs-lookup"><span data-stu-id="247a0-103">Compiler Warning (level 2) CS0440</span></span>
<span data-ttu-id="247a0-104">Definování aliasu s názvem Global se nedoporučuje, protože Global:: vždycky odkazuje na globální obor názvů, ne na alias.</span><span class="sxs-lookup"><span data-stu-id="247a0-104">Defining an alias named 'global' is ill-advised since 'global::' always references the global namespace and not an alias</span></span>  
  
 <span data-ttu-id="247a0-105">Toto upozornění se vydá při definování aliasu s názvem Global.</span><span class="sxs-lookup"><span data-stu-id="247a0-105">This warning is issued when you define an alias named global.</span></span>  
  
## <a name="example"></a><span data-ttu-id="247a0-106">Příklad</span><span class="sxs-lookup"><span data-stu-id="247a0-106">Example</span></span>  
 <span data-ttu-id="247a0-107">Následující příklad generuje CS0440:</span><span class="sxs-lookup"><span data-stu-id="247a0-107">The following example generates CS0440:</span></span>  
  
```csharp  
// CS0440.cs  
// Compile with: /W:1  
  
using global = MyClass;   // CS0440  
class MyClass  
{  
    static void Main()  
    {  
        // Note how global refers to the global namespace  
        // even though it is redefined above.  
        global::System.Console.WriteLine();  
    }  
}  
```
