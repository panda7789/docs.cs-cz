# <a name="choosing-between-class-and-struct"></a><span data-ttu-id="6d734-102">Volba mezi třídou a strukturou</span><span class="sxs-lookup"><span data-stu-id="6d734-102">Choosing Between Class and Struct</span></span>
<span data-ttu-id="6d734-103">Jedno ze základních rozhodnutí o návrhu každé plošky návrháře architektury je bez ohledu na to, zda je možné navrhnout typ jako třídu (odkazový typ) nebo jako strukturu (typ hodnoty).</span><span class="sxs-lookup"><span data-stu-id="6d734-103">One of the basic design decisions every framework designer faces is whether to design a type as a class (a reference type) or as a struct (a value type).</span></span> <span data-ttu-id="6d734-104">Dobrým porozumění rozdílům v chování typů odkazu a hodnot je rozhodující při této volbě.</span><span class="sxs-lookup"><span data-stu-id="6d734-104">Good understanding of the differences in the behavior of reference types and value types is crucial in making this choice.</span></span>  
  
 <span data-ttu-id="6d734-105">První rozdíl mezi typy odkazů a typy hodnot, které budeme brát v úvahu, je, že typy odkazů jsou přiděleny na haldě a uvolňování paměti, zatímco typy hodnot jsou přiděleny buď v zásobníku, nebo vloženy v případě, že je zásobník unwind nebo při zrušení přidělení jejich nadřazeného typu.</span><span class="sxs-lookup"><span data-stu-id="6d734-105">The first difference between reference types and value types we will consider is that reference types are allocated on the heap and garbage-collected, whereas value types are allocated either on the stack or inline in containing types and deallocated when the stack unwinds or when their containing type gets deallocated.</span></span> <span data-ttu-id="6d734-106">Proto jsou alokace a navracení hodnotových typů všeobecně levnější než přidělení a zrušení přidělení typů odkazů.</span><span class="sxs-lookup"><span data-stu-id="6d734-106">Therefore, allocations and deallocations of value types are in general cheaper than allocations and deallocations of reference types.</span></span>  
  
 <span data-ttu-id="6d734-107">Dále jsou pole typů odkazů rozdělena mimo řádek, což znamená, že prvky pole jsou pouze odkazy na instance typu odkazu, který je umístěn na haldě.</span><span class="sxs-lookup"><span data-stu-id="6d734-107">Next, arrays of reference types are allocated out-of-line, meaning the array elements are just references to instances of the reference type residing on the heap.</span></span> <span data-ttu-id="6d734-108">Pole hodnot typu jsou přidělena jako vložená, což znamená, že prvky pole jsou skutečnými instancemi typu hodnoty.</span><span class="sxs-lookup"><span data-stu-id="6d734-108">Value type arrays are allocated inline, meaning that the array elements are the actual instances of the value type.</span></span> <span data-ttu-id="6d734-109">Proto jsou přidělení a dealokace polí hodnotových typů mnohem levnější než přidělení a zrušení přidělení polí typu odkazu.</span><span class="sxs-lookup"><span data-stu-id="6d734-109">Therefore, allocations and deallocations of value type arrays are much cheaper than allocations and deallocations of reference type arrays.</span></span> <span data-ttu-id="6d734-110">Kromě toho ve většině případů typ hodnoty pole představuje mnohem lepší rozsah odkazů.</span><span class="sxs-lookup"><span data-stu-id="6d734-110">In addition, in a majority of cases value type arrays exhibit much better locality of reference.</span></span>  
  
 <span data-ttu-id="6d734-111">Další rozdíl se týká využití paměti.</span><span class="sxs-lookup"><span data-stu-id="6d734-111">The next difference is related to memory usage.</span></span> <span data-ttu-id="6d734-112">Typy hodnot se při přetypování na typ odkazu nebo na jedno z rozhraní, které implementují, vrátí do pole.</span><span class="sxs-lookup"><span data-stu-id="6d734-112">Value types get boxed when cast to a reference type or one of the interfaces they implement.</span></span> <span data-ttu-id="6d734-113">Po přetypování zpátky na typ hodnoty se rozstanou.</span><span class="sxs-lookup"><span data-stu-id="6d734-113">They get unboxed when cast back to the value type.</span></span> <span data-ttu-id="6d734-114">Vzhledem k tomu, že pole jsou objekty, které jsou přiděleny haldě a jsou uvolněny do paměti, příliš mnoho zabalení a rozbalení může mít negativní vliv na haldu, uvolňování paměti a nakonec výkon aplikace.</span><span class="sxs-lookup"><span data-stu-id="6d734-114">Because boxes are objects that are allocated on the heap and are garbage-collected, too much boxing and unboxing can have a negative impact on the heap, the garbage collector, and ultimately the performance of the application.</span></span>  <span data-ttu-id="6d734-115">Naopak žádné takové zabalení nevzniká jako přetypování typů odkazů.</span><span class="sxs-lookup"><span data-stu-id="6d734-115">In contrast, no such boxing occurs as reference types are cast.</span></span> <span data-ttu-id="6d734-116">(Další informace naleznete v tématu [zabalení a rozbalení](../../csharp/programming-guide/types/boxing-and-unboxing.md)).</span><span class="sxs-lookup"><span data-stu-id="6d734-116">(For more information, see [Boxing and Unboxing](../../csharp/programming-guide/types/boxing-and-unboxing.md)).</span></span>
  
 <span data-ttu-id="6d734-117">V dalším kroku přiřazení typu odkazu kopírují odkaz, zatímco přiřazení typu hodnoty kopírují celou hodnotu.</span><span class="sxs-lookup"><span data-stu-id="6d734-117">Next, reference type assignments copy the reference, whereas value type assignments copy the entire value.</span></span> <span data-ttu-id="6d734-118">Proto přiřazení velkých typů odkazů jsou levnější než přiřazení typů velkých hodnot.</span><span class="sxs-lookup"><span data-stu-id="6d734-118">Therefore, assignments of large reference types are cheaper than assignments of large value types.</span></span>  
  
 <span data-ttu-id="6d734-119">Nakonec odkazové typy jsou předány odkazem, zatímco typy hodnot jsou předány hodnotou.</span><span class="sxs-lookup"><span data-stu-id="6d734-119">Finally, reference types are passed by reference, whereas value types are passed by value.</span></span> <span data-ttu-id="6d734-120">Změny instance typu odkazu ovlivňují všechny odkazy odkazující na instanci.</span><span class="sxs-lookup"><span data-stu-id="6d734-120">Changes to an instance of a reference type affect all references pointing to the instance.</span></span> <span data-ttu-id="6d734-121">Instance typu hodnoty jsou zkopírovány, když jsou předány hodnotou.</span><span class="sxs-lookup"><span data-stu-id="6d734-121">Value type instances are copied when they are passed by value.</span></span> <span data-ttu-id="6d734-122">Když dojde ke změně instance hodnotového typu, samozřejmě nemá vliv na žádnou z jeho kopií.</span><span class="sxs-lookup"><span data-stu-id="6d734-122">When an instance of a value type is changed, it of course does not affect any of its copies.</span></span> <span data-ttu-id="6d734-123">Vzhledem k tomu, že kopie nejsou vytvořeny explicitně uživatelem, ale jsou implicitně vytvořeny, když jsou předány argumenty nebo vrátí návratové hodnoty, mohou být typy hodnot, které lze změnit, matoucí pro mnoho uživatelů.</span><span class="sxs-lookup"><span data-stu-id="6d734-123">Because the copies are not created explicitly by the user but are implicitly created when arguments are passed or return values are returned, value types that can be changed can be confusing to many users.</span></span> <span data-ttu-id="6d734-124">Proto by měly být typy hodnot neměnné.</span><span class="sxs-lookup"><span data-stu-id="6d734-124">Therefore, value types should be immutable.</span></span>  
  
 <span data-ttu-id="6d734-125">Jako pravidlo pro palec by většina typů v rozhraní měla být třídy.</span><span class="sxs-lookup"><span data-stu-id="6d734-125">As a rule of thumb, the majority of types in a framework should be classes.</span></span> <span data-ttu-id="6d734-126">Existují však některé situace, ve kterých charakteristiky typu hodnoty usnadňují používání struktur.</span><span class="sxs-lookup"><span data-stu-id="6d734-126">There are, however, some situations in which the characteristics of a value type make it more appropriate to use structs.</span></span>  
  
 <span data-ttu-id="6d734-127">**✓ CONSIDER** definice struktury místo třídu, pokud instance typu jsou malé a běžně krátkodobou nebo jsou běžně součástí jiné objekty.</span><span class="sxs-lookup"><span data-stu-id="6d734-127">**✓ CONSIDER** defining a struct instead of a class if instances of the type are small and commonly short-lived or are commonly embedded in other objects.</span></span>  
  
 <span data-ttu-id="6d734-128">**X AVOID** definice struktury, pokud má tento typ všechny následující vlastnosti:</span><span class="sxs-lookup"><span data-stu-id="6d734-128">**X AVOID** defining a struct unless the type has all of the following characteristics:</span></span>  
  
- <span data-ttu-id="6d734-129">Logicky představuje jedinou hodnotu, podobně jako primitivní typy (`int`, `double`atd.).</span><span class="sxs-lookup"><span data-stu-id="6d734-129">It logically represents a single value, similar to primitive types (`int`, `double`, etc.).</span></span>  
  
- <span data-ttu-id="6d734-130">Má velikost instance 16 bajtů.</span><span class="sxs-lookup"><span data-stu-id="6d734-130">It has an instance size under 16 bytes.</span></span>  
  
- <span data-ttu-id="6d734-131">Je neměnný.</span><span class="sxs-lookup"><span data-stu-id="6d734-131">It is immutable.</span></span>  
  
- <span data-ttu-id="6d734-132">Nemusí být často ohraničená.</span><span class="sxs-lookup"><span data-stu-id="6d734-132">It will not have to be boxed frequently.</span></span>  
  
 <span data-ttu-id="6d734-133">Ve všech ostatních případech byste měli definovat typy jako třídy.</span><span class="sxs-lookup"><span data-stu-id="6d734-133">In all other cases, you should define your types as classes.</span></span>
