# <a name="base-classes-for-implementing-abstractions"></a><span data-ttu-id="3dc8e-102">Základní třídy pro implementaci abstrakcí</span><span class="sxs-lookup"><span data-stu-id="3dc8e-102">Base Classes for Implementing Abstractions</span></span>
<span data-ttu-id="3dc8e-103">V případě, že je z něj odvozena jiná třída, třída se bude zcela vymluvit jako základní třída.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-103">Strictly speaking, a class becomes a base class when another class is derived from it.</span></span> <span data-ttu-id="3dc8e-104">Pro účely této části je však základní třídou třída navržená hlavně pro poskytnutí společné abstrakce nebo pro jiné třídy pro opakované použití některé výchozí implementace, i když je dědění.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-104">For the purpose of this section, however, a base class is a class designed mainly to provide a common abstraction or for other classes to reuse some default implementation though inheritance.</span></span> <span data-ttu-id="3dc8e-105">Základní třídy obvykle sedí uprostřed hierarchií dědičnosti mezi abstrakcí v kořenovém adresáři hierarchie a několika vlastními implementacemi v dolní části.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-105">Base classes usually sit in the middle of inheritance hierarchies, between an abstraction at the root of a hierarchy and several custom implementations at the bottom.</span></span>  
  
 <span data-ttu-id="3dc8e-106">Slouží jako implementační pomocníky k implementaci abstrakcí.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-106">They serve as implementation helpers for implementing abstractions.</span></span> <span data-ttu-id="3dc8e-107">Například jedna z abstrakcí rozhraní pro uspořádané kolekce položek je <xref:System.Collections.Generic.IList%601> rozhraní.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-107">For example, one of the Framework’s abstractions for ordered collections of items is the <xref:System.Collections.Generic.IList%601> interface.</span></span> <span data-ttu-id="3dc8e-108">Implementace <xref:System.Collections.Generic.IList%601> není triviální, a proto rozhraní poskytuje několik základních tříd, jako je například <xref:System.Collections.ObjectModel.Collection%601> a <xref:System.Collections.ObjectModel.KeyedCollection%602>, které slouží jako pomocník pro implementaci vlastních kolekcí.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-108">Implementing <xref:System.Collections.Generic.IList%601> is not trivial, and therefore the Framework provides several base classes, such as <xref:System.Collections.ObjectModel.Collection%601> and <xref:System.Collections.ObjectModel.KeyedCollection%602>, which serve as helpers for implementing custom collections.</span></span>  
  
 <span data-ttu-id="3dc8e-109">Základní třídy většinou nejsou vhodné k tomu, aby sloužily jako abstrakce samotných, protože by měly obsahovat příliš mnoho implementace.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-109">Base classes are usually not suited to serve as abstractions by themselves, because they tend to contain too much implementation.</span></span> <span data-ttu-id="3dc8e-110">Například základní třída `Collection<T>` obsahuje mnoho implementace souvisejících se skutečností, že implementuje neobecné `IList` rozhraní (pro integraci lépe s neobecnými kolekcemi) a fakt, že se jedná o kolekci položek uložených v paměti v jednom z jeho polí.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-110">For example, the `Collection<T>` base class contains lots of implementation related to the fact that it implements the nongeneric `IList` interface (to integrate better with nongeneric collections) and to the fact that it is a collection of items stored in memory in one of its fields.</span></span>  
  
 <span data-ttu-id="3dc8e-111">Jak už bylo popsáno, základní třídy mohou poskytnout nevýznamnou nápovědu pro uživatele, kteří potřebují implementovat abstrakce, ale ve stejnou dobu se může jednat o významnou zodpovědnost.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-111">As previously discussed, base classes can provide invaluable help for users who need to implement abstractions, but at the same time they can be a significant liability.</span></span> <span data-ttu-id="3dc8e-112">Přidávají oblast Surface a zvyšují hloubku hierarchií dědičnosti a tak koncepčně komplikuje rozhraní.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-112">They add surface area and increase the depth of inheritance hierarchies and so conceptually complicate the framework.</span></span> <span data-ttu-id="3dc8e-113">Proto by měly být použity základní třídy pouze v případě, že poskytují významnou hodnotu uživatelům rozhraní.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-113">Therefore, base classes should be used only if they provide significant value to the users of the framework.</span></span> <span data-ttu-id="3dc8e-114">Měli byste se vyhnout, pokud poskytují hodnotu pouze implementátorům rozhraní, v takovém případě by mělo být silně zváženo delegování na vnitřní implementaci místo dědičnosti základní třídy.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-114">They should be avoided if they provide value only to the implementers of the framework, in which case delegation to an internal implementation instead of inheritance from a base class should be strongly considered.</span></span>  
  
 <span data-ttu-id="3dc8e-115">**✓ CONSIDER** provedení základní třídy abstraktní i v případě, že neobsahují žádné abstraktní členy.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-115">**✓ CONSIDER** making base classes abstract even if they don’t contain any abstract members.</span></span> <span data-ttu-id="3dc8e-116">To jasně oznamuje uživatelům, že třída je navržena výhradně pro dědění.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-116">This clearly communicates to the users that the class is designed solely to be inherited from.</span></span>  
  
 <span data-ttu-id="3dc8e-117">**✓ CONSIDER** umístění základních tříd do samostatného oboru názvů z nejdůležitějších scénář typů.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-117">**✓ CONSIDER** placing base classes in a separate namespace from the mainline scenario types.</span></span> <span data-ttu-id="3dc8e-118">Podle definice jsou základní třídy určené pro pokročilé scénáře rozšiřitelnosti, proto nejsou zajímavé pro většinu uživatelů.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-118">By definition, base classes are intended for advanced extensibility scenarios and therefore are not interesting to the majority of users.</span></span>  
  
 <span data-ttu-id="3dc8e-119">**X AVOID** pojmenování základní třídy s příponou "Základní", pokud třída je určena pro použití v veřejná rozhraní API.</span><span class="sxs-lookup"><span data-stu-id="3dc8e-119">**X AVOID** naming base classes with a "Base" suffix if the class is intended for use in public APIs.</span></span>
