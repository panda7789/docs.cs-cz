# <a name="property-design"></a><span data-ttu-id="8b1ba-102">Návrh vlastnosti</span><span class="sxs-lookup"><span data-stu-id="8b1ba-102">Property Design</span></span>
<span data-ttu-id="8b1ba-103">I když jsou vlastnosti technicky velmi podobné metodám, jsou v souladu s jejich scénáři použití poměrně odlišné.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="8b1ba-104">Měly by se zobrazit jako inteligentní pole.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-104">They should be seen as smart fields.</span></span> <span data-ttu-id="8b1ba-105">Mají volání syntaxe polí a flexibilitu metod.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>

 <span data-ttu-id="8b1ba-106">✔️ vytvořit vlastnosti jen pro získání, pokud volající by neměl být schopný změnit hodnotu vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-106">✔️ DO create get-only properties if the caller should not be able to change the value of the property.</span></span>

 <span data-ttu-id="8b1ba-107">Pamatujte, že pokud je typ vlastnosti proměnlivý typ odkazu, hodnota vlastnosti může být změněna i v případě, že je vlastnost určena pouze pro čtení.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>

 <span data-ttu-id="8b1ba-108">❌ neposkytují vlastnosti nebo vlastnosti jen pro nastavení s vlastností setter širší přístupnost než getter.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-108">❌ DO NOT provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>

 <span data-ttu-id="8b1ba-109">Nepoužívejte například vlastnosti s veřejnou setter a Protected getter.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-109">For example, do not use properties with a public setter and a protected getter.</span></span>

 <span data-ttu-id="8b1ba-110">Pokud vlastnost getter nelze poskytnout, implementujte funkci namísto metody.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="8b1ba-111">Zvažte možnost spustit název metody s `Set` a sledujte, co byste měli pojmenovat vlastnost.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="8b1ba-112">Například <xref:System.AppDomain> má metodu nazvanou `SetCachePath` místo toho, aby byla vlastnost pouze set s názvem `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>

 <span data-ttu-id="8b1ba-113">✔️ poskytují výchozí hodnoty rozumné pro všechny vlastnosti, což zajistí, že výchozí hodnoty nebudou mít za následek bezpečnostní riziko ani vážně selhalo neefektivní kód.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-113">✔️ DO provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>

 <span data-ttu-id="8b1ba-114">✔️ Povolit nastavení vlastností v libovolném pořadí, i když dojde k dočasnému neplatnému stavu objektu.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-114">✔️ DO allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>

 <span data-ttu-id="8b1ba-115">Je běžné, že dvě nebo více vlastností bude možné vzájemně souviset s bodem, kde některé hodnoty jedné vlastnosti mohou být neplatné vzhledem k hodnotám jiných vlastností stejného objektu.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="8b1ba-116">V takových případech by měly být výjimky, které jsou výsledkem neplatného stavu, odloženy, dokud se tyto vlastnosti nepoužívají společně s objektem.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>

 <span data-ttu-id="8b1ba-117">✔️ zachovat předchozí hodnotu, pokud vlastnost setter vyvolá výjimku.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-117">✔️ DO preserve the previous value if a property setter throws an exception.</span></span>

 <span data-ttu-id="8b1ba-118">❌ Vyhněte se vyvolávání výjimek z mechanismů getter vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-118">❌ AVOID throwing exceptions from property getters.</span></span>

 <span data-ttu-id="8b1ba-119">Metody getter vlastnosti by měly být jednoduché operace a neměly by obsahovat žádné předběžné podmínky.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="8b1ba-120">Pokud getter může vyvolat výjimku, měla by být pravděpodobně přepracována jako metoda.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="8b1ba-121">Všimněte si, že toto pravidlo se nevztahuje na indexery, kde očekáváme výjimky v důsledku ověřování argumentů.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>

### <a name="indexed-property-design"></a><span data-ttu-id="8b1ba-122">Návrh indexovaných vlastností</span><span class="sxs-lookup"><span data-stu-id="8b1ba-122">Indexed Property Design</span></span>
 <span data-ttu-id="8b1ba-123">Indexovaná vlastnost je speciální vlastnost, která může mít parametry a může být volána se speciální syntaxí podobnou indexování pole.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>

 <span data-ttu-id="8b1ba-124">Indexované vlastnosti se běžně označují jako indexery.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="8b1ba-125">Indexery by měly být používány pouze v rozhraních API, která poskytují přístup k položkám v logické kolekci.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="8b1ba-126">Například řetězec je kolekce znaků a indexer na <xref:System.String?displayProperty=nameWithType> byl přidán pro přístup ke svým znakům.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>

 <span data-ttu-id="8b1ba-127">✔️ Zvažte použití indexerů k poskytnutí přístupu k datům uloženým v interním poli.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-127">✔️ CONSIDER using indexers to provide access to data stored in an internal array.</span></span>

 <span data-ttu-id="8b1ba-128">✔️ Zvažte poskytování indexerů na typech představujících kolekce položek.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-128">✔️ CONSIDER providing indexers on types representing collections of items.</span></span>

 <span data-ttu-id="8b1ba-129">❌ se vyhnout použití indexovaných vlastností s více než jedním parametrem.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-129">❌ AVOID using indexed properties with more than one parameter.</span></span>

 <span data-ttu-id="8b1ba-130">Pokud návrh vyžaduje více parametrů, převezměte v úvahu, zda vlastnost ve skutečnosti představuje přistupující objekt k logické kolekci.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="8b1ba-131">Pokud ne, použijte místo toho metody.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-131">If it does not, use methods instead.</span></span> <span data-ttu-id="8b1ba-132">Zvažte možnost spustit název metody pomocí `Get` nebo `Set`.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-132">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="8b1ba-133">❌ zabránit indexerům s jinými typy parametrů než <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>nebo Enum.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-133">❌ AVOID indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>

 <span data-ttu-id="8b1ba-134">Pokud návrh vyžaduje jiné typy parametrů, silně znovu Vyhodnoťte, jestli rozhraní API skutečně představuje přistupující objekt k logické kolekci.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="8b1ba-135">Pokud tomu tak není, použijte metodu.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-135">If it does not, use a method.</span></span> <span data-ttu-id="8b1ba-136">Zvažte možnost spustit název metody pomocí `Get` nebo `Set`.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-136">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="8b1ba-137">✔️ použít název `Item` u indexovaných vlastností, pokud neexistuje zjevně lepší název (například, viz vlastnost <xref:System.String.Chars%2A> na `System.String`).</span><span class="sxs-lookup"><span data-stu-id="8b1ba-137">✔️ DO use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>

 <span data-ttu-id="8b1ba-138">V C#nástroji jsou indexery ve výchozím nastavení pojmenované položky.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="8b1ba-139">K přizpůsobení tohoto názvu lze použít <xref:System.Runtime.CompilerServices.IndexerNameAttribute>.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>

 <span data-ttu-id="8b1ba-140">❌ neposkytují indexer a metody, které jsou sémanticky ekvivalentní.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-140">❌ DO NOT provide both an indexer and methods that are semantically equivalent.</span></span>

 <span data-ttu-id="8b1ba-141">❌ neposkytují více než jednu skupinu přetížených indexerů v jednom typu.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-141">❌ DO NOT provide more than one family of overloaded indexers in one type.</span></span>

 <span data-ttu-id="8b1ba-142">Toto je vynutil C# kompilátorem.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-142">This is enforced by the C# compiler.</span></span>

 <span data-ttu-id="8b1ba-143">❌ nepoužívat výchozí indexované vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-143">❌ DO NOT use nondefault indexed properties.</span></span>

 <span data-ttu-id="8b1ba-144">Toto je vynutil C# kompilátorem.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-144">This is enforced by the C# compiler.</span></span>

### <a name="property-change-notification-events"></a><span data-ttu-id="8b1ba-145">Události upozornění na změnu vlastnosti</span><span class="sxs-lookup"><span data-stu-id="8b1ba-145">Property Change Notification Events</span></span>
 <span data-ttu-id="8b1ba-146">Někdy je užitečné poskytnout událost upozorňující uživatele na změny v hodnotě vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="8b1ba-147">Například `System.Windows.Forms.Control` vyvolává událost `TextChanged` po změně hodnoty vlastnosti `Text`.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>

 <span data-ttu-id="8b1ba-148">✔️ Zvažte vyvolání události oznámení o změně při změně hodnot vlastností v rozhraních API na vysoké úrovni (obvykle komponent návrháře).</span><span class="sxs-lookup"><span data-stu-id="8b1ba-148">✔️ CONSIDER raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>

 <span data-ttu-id="8b1ba-149">Pokud je dobrým scénářem, že uživatel ví, kdy se změní vlastnost objektu, měl by objekt vyvolat událost upozornění na změnu pro vlastnost.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>

 <span data-ttu-id="8b1ba-150">Nepředpokládá se však, že bude nutné zvýšit nároky na vyvolání takových událostí pro rozhraní API nižší úrovně, jako jsou základní typy nebo kolekce.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="8b1ba-151"><xref:System.Collections.Generic.List%601> například nevyvolává takové události, když se do seznamu přidá nová položka a změní se vlastnost `Count`.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>

 <span data-ttu-id="8b1ba-152">✔️ Zvažte vyvolání události oznámení o změně při změně hodnoty vlastnosti prostřednictvím externích sil.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-152">✔️ CONSIDER raising change notification events when the value of a property changes via external forces.</span></span>

 <span data-ttu-id="8b1ba-153">Pokud se hodnota vlastnosti mění přes některé externí síly (způsobem, který je jiný než voláním metod objektu), vyvolávají události upozorňující na vývojáře, že se hodnota mění a změnila.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="8b1ba-154">Dobrým příkladem je vlastnost `Text` ovládacího prvku textového pole.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="8b1ba-155">Když uživatel zadá text do `TextBox`, hodnota vlastnosti se automaticky změní.</span><span class="sxs-lookup"><span data-stu-id="8b1ba-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
