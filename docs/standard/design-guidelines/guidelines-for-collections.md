 <span data-ttu-id="97e73-170">**✓ CONSIDER** dědění z `Collection<T>`, `ReadOnlyCollection<T>`, nebo `KeyedCollection<TKey,TItem>` při návrhu nové kolekce.</span><span class="sxs-lookup"><span data-stu-id="97e73-170">**✓ CONSIDER** inheriting from `Collection<T>`, `ReadOnlyCollection<T>`, or `KeyedCollection<TKey,TItem>` when designing new collections.</span></span>  
  
 <span data-ttu-id="97e73-171">**✓ DO** implementovat `IEnumerable<T>` při návrhu nové kolekce.</span><span class="sxs-lookup"><span data-stu-id="97e73-171">**✓ DO** implement `IEnumerable<T>` when designing new collections.</span></span> <span data-ttu-id="97e73-172">Zvažte implementaci `ICollection<T>` nebo dokonce `IList<T>`, kde to dává smysl.</span><span class="sxs-lookup"><span data-stu-id="97e73-172">Consider implementing `ICollection<T>` or even `IList<T>` where it makes sense.</span></span>  
  
 <span data-ttu-id="97e73-173">Při implementaci takové vlastní kolekce se řiďte vzorem rozhraní API vytvořeným `Collection<T>` a `ReadOnlyCollection<T>` co nejpřesněji.</span><span class="sxs-lookup"><span data-stu-id="97e73-173">When implementing such custom collection, follow the API pattern established by `Collection<T>` and `ReadOnlyCollection<T>` as closely as possible.</span></span> <span data-ttu-id="97e73-174">To znamená, že je třeba implementovat stejné členy explicitně, pojmenovat parametry jako tyto dvě kolekce a tak dále.</span><span class="sxs-lookup"><span data-stu-id="97e73-174">That is, implement the same members explicitly, name the parameters like these two collections name them, and so on.</span></span>  
  
 <span data-ttu-id="97e73-175">**✓ CONSIDER** implementace rozhraní neobecné kolekce (`IList` a `ICollection`) Pokud kolekce se často předá rozhraní API trvá tato rozhraní jako vstup.</span><span class="sxs-lookup"><span data-stu-id="97e73-175">**✓ CONSIDER** implementing nongeneric collection interfaces (`IList` and `ICollection`) if the collection will often be passed to APIs taking these interfaces as input.</span></span>  
  
 <span data-ttu-id="97e73-176">**X AVOID** implementace rozhraní pro kolekce na typy s komplexní rozhraní API koncept kolekce, které nejsou.</span><span class="sxs-lookup"><span data-stu-id="97e73-176">**X AVOID** implementing collection interfaces on types with complex APIs unrelated to the concept of a collection.</span></span>

### <a name="naming-custom-collections"></a><span data-ttu-id="97e73-179">Pojmenovávání vlastních kolekcí</span><span class="sxs-lookup"><span data-stu-id="97e73-179">Naming Custom Collections</span></span>  
 <span data-ttu-id="97e73-180">Kolekce (typy, které implementují `IEnumerable`) se vytvářejí hlavně ze dvou důvodů: (1) pro vytvoření nové struktury dat s operacemi specifickými pro strukturu a často různé charakteristiky výkonu než stávající datové struktury (např. <xref:System.Collections.Generic.List%601>, <xref:System.Collections.Generic.LinkedList%601>, <xref:System.Collections.Generic.Stack%601>) a (2) pro vytvoření specializované kolekce pro uchovávání konkrétní sady položek (například <xref:System.Collections.Specialized.StringCollection>).</span><span class="sxs-lookup"><span data-stu-id="97e73-180">Collections (types that implement `IEnumerable`) are created mainly for two reasons: (1) to create a new data structure with structure-specific operations and often different performance characteristics than existing data structures (e.g.,  <xref:System.Collections.Generic.List%601>, <xref:System.Collections.Generic.LinkedList%601>, <xref:System.Collections.Generic.Stack%601>), and (2) to create a specialized collection for holding a specific set of items (e.g.,  <xref:System.Collections.Specialized.StringCollection>).</span></span> <span data-ttu-id="97e73-181">V interní implementaci aplikací a knihoven se často používají datové struktury.</span><span class="sxs-lookup"><span data-stu-id="97e73-181">Data structures are most often used in the internal implementation of applications and libraries.</span></span> <span data-ttu-id="97e73-182">Specializované kolekce jsou převážně vystaveny v rozhraních API (jako typy vlastností a parametrů).</span><span class="sxs-lookup"><span data-stu-id="97e73-182">Specialized collections are mainly to be exposed in APIs (as property and parameter types).</span></span>  
  
 <span data-ttu-id="97e73-183">**✓ DO** používat příponu "Slovník" v názvech abstrakce implementace `IDictionary` nebo `IDictionary<TKey,TValue>`.</span><span class="sxs-lookup"><span data-stu-id="97e73-183">**✓ DO** use the "Dictionary" suffix in names of abstractions implementing `IDictionary` or `IDictionary<TKey,TValue>`.</span></span>  
  
 <span data-ttu-id="97e73-184">**✓ DO** používat příponu "Kolekce" v názvech typů implementace `IEnumerable` (nebo některého z jejich potomků) a představuje seznam položek.</span><span class="sxs-lookup"><span data-stu-id="97e73-184">**✓ DO** use the "Collection" suffix in names of types implementing `IEnumerable` (or any of its descendants) and representing a list of items.</span></span>  
  
 <span data-ttu-id="97e73-185">**✓ DO** použijte název struktura příslušná data pro vlastní datové struktury.</span><span class="sxs-lookup"><span data-stu-id="97e73-185">**✓ DO** use the appropriate data structure name for custom data structures.</span></span>  
  
 <span data-ttu-id="97e73-186">**X AVOID** pomocí libovolné přípony zdání konkrétní implementace, jako je například "LinkedList" nebo "Zatřiďovací tabulky," v názvech abstrakce kolekce.</span><span class="sxs-lookup"><span data-stu-id="97e73-186">**X AVOID** using any suffixes implying particular implementation, such as "LinkedList" or "Hashtable," in names of collection abstractions.</span></span>  
  
 <span data-ttu-id="97e73-187">**✓ CONSIDER** prefixu názvy kolekci s názvem typu položky.</span><span class="sxs-lookup"><span data-stu-id="97e73-187">**✓ CONSIDER** prefixing collection names with the name of the item type.</span></span> <span data-ttu-id="97e73-188">Například kolekce, která ukládá položky typu `Address` (implementace `IEnumerable<Address>`), by měla být pojmenována `AddressCollection`.</span><span class="sxs-lookup"><span data-stu-id="97e73-188">For example, a collection storing items of type `Address` (implementing `IEnumerable<Address>`) should be named `AddressCollection`.</span></span> <span data-ttu-id="97e73-189">Pokud je typ položky rozhraní, může být předpona "I" typu položky vynechána.</span><span class="sxs-lookup"><span data-stu-id="97e73-189">If the item type is an interface, the "I" prefix of the item type can be omitted.</span></span> <span data-ttu-id="97e73-190">Proto může být kolekce <xref:System.IDisposable>ch položek volána `DisposableCollection`.</span><span class="sxs-lookup"><span data-stu-id="97e73-190">Thus, a collection of <xref:System.IDisposable> items can be called `DisposableCollection`.</span></span>  
  
 <span data-ttu-id="97e73-191">**✓ CONSIDER** pomocí předpony "Jen pro čtení" v názvech kolekcí jen pro čtení, pokud odpovídající kolekci lze zapisovat, mohou být přidány nebo již existuje v rámci.</span><span class="sxs-lookup"><span data-stu-id="97e73-191">**✓ CONSIDER** using the "ReadOnly" prefix in names of read-only collections if a corresponding writeable collection might be added or already exists in the framework.</span></span>  
  
 <span data-ttu-id="97e73-192">Například kolekce řetězců jen pro čtení by měla být volána `ReadOnlyStringCollection`.</span><span class="sxs-lookup"><span data-stu-id="97e73-192">For example, a read-only collection of strings should be called `ReadOnlyStringCollection`.</span></span>
