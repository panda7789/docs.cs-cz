# <a name="assembly-names"></a><span data-ttu-id="c2463-103">Názvy sestavení</span><span class="sxs-lookup"><span data-stu-id="c2463-103">Assembly names</span></span>

> [!NOTE]
> <span data-ttu-id="c2463-109">Do identity sestavení v .NET Framework verze 2,0 se přidá architektura procesoru, aby se povolily verze sestavení specifické pro procesor.</span><span class="sxs-lookup"><span data-stu-id="c2463-109">Processor architecture is added to the assembly identity in the .NET Framework version 2.0, to allow processor-specific versions of assemblies.</span></span> <span data-ttu-id="c2463-110">Můžete vytvořit verze sestavení, jehož identita se liší pouze architekturou procesoru, například 32-bit a 64-bit specifických verzí procesoru.</span><span class="sxs-lookup"><span data-stu-id="c2463-110">You can create versions of an assembly whose identity differs only by processor architecture, for example 32-bit and 64-bit processor-specific versions.</span></span> <span data-ttu-id="c2463-111">Architektura procesoru není pro silné názvy nutná.</span><span class="sxs-lookup"><span data-stu-id="c2463-111">Processor architecture is not required for strong names.</span></span> <span data-ttu-id="c2463-112">Další informace naleznete v tématu <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="c2463-112">For more information, see <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span></span>

 <span data-ttu-id="c2463-113">V tomto příkladu plně kvalifikovaný název označuje, že `myTypes` sestavení má silný název s tokenem veřejného klíče, má hodnotu jazykové verze pro americkou angličtinu a má číslo verze 1.0.1234.0.</span><span class="sxs-lookup"><span data-stu-id="c2463-113">In this example, the fully qualified name indicates that the `myTypes` assembly has a strong name with a public key token, has the culture value for US English, and has a version number of 1.0.1234.0.</span></span> <span data-ttu-id="c2463-114">Jeho architektura procesoru je "MSIL", což znamená, že bude za běhu (JIT) zkompilován do 32 bitového kódu nebo 64 kódu v závislosti na operačním systému a procesoru.</span><span class="sxs-lookup"><span data-stu-id="c2463-114">Its processor architecture is "msil", which means that it will be just-in-time (JIT)-compiled to 32-bit code or 64-bit code depending on the operating system and processor.</span></span>

 <span data-ttu-id="c2463-115">Kód, který požaduje typy v sestavení, musí používat plně kvalifikovaný název sestavení.</span><span class="sxs-lookup"><span data-stu-id="c2463-115">Code that requests types in an assembly must use a fully qualified assembly name.</span></span> <span data-ttu-id="c2463-116">Tato metoda se nazývá plně kvalifikovaná vazba.</span><span class="sxs-lookup"><span data-stu-id="c2463-116">This is called fully qualified binding.</span></span> <span data-ttu-id="c2463-117">Částečná vazba, která určuje pouze název sestavení, není povolena při odkazování na sestavení v .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="c2463-117">Partial binding, which specifies only an assembly name, is not permitted when referencing assemblies in the .NET Framework.</span></span>

 <span data-ttu-id="c2463-118">Všechny odkazy na sestavení, které tvoří .NET Framework, musí také obsahovat plně kvalifikovaný název sestavení.</span><span class="sxs-lookup"><span data-stu-id="c2463-118">All assembly references to assemblies that make up the .NET Framework must also contain the fully qualified name of the assembly.</span></span> <span data-ttu-id="c2463-119">Například odkaz na sestavení System. data .NET Framework pro verzi 1,0 by zahrnoval:</span><span class="sxs-lookup"><span data-stu-id="c2463-119">For example, a reference to the System.Data .NET Framework assembly for version 1.0 would include:</span></span>

```
System.data, version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089
```

 <span data-ttu-id="c2463-120">Všimněte si, že verze odpovídá číslu verze všech .NET Framework sestavení, která byla dodávána s .NET Framework verze 1,0.</span><span class="sxs-lookup"><span data-stu-id="c2463-120">Note that the version corresponds to the version number of all .NET Framework assemblies that shipped with .NET Framework version 1.0.</span></span> <span data-ttu-id="c2463-121">Pro .NET Framework sestavení je hodnota jazykové verze vždycky neutrální a veřejný klíč je stejný, jak je uvedeno v předchozím příkladu.</span><span class="sxs-lookup"><span data-stu-id="c2463-121">For .NET Framework assemblies, the culture value is always neutral, and the public key is the same as shown in the above example.</span></span>

 <span data-ttu-id="c2463-122">Chcete-li například přidat odkaz na sestavení do konfiguračního souboru pro nastavení naslouchacího procesu trasování, zahrnete plně kvalifikovaný název systémového .NET Framework sestavení:</span><span class="sxs-lookup"><span data-stu-id="c2463-122">For example, to add an assembly reference in a configuration file to set up a trace listener, you would include the fully qualified name of the system .NET Framework assembly:</span></span>

```xml
<add name="myListener" type="System.Diagnostics.TextWriterTraceListener, System, Version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" initializeData="c:\myListener.log" />
```

> [!NOTE]
> <span data-ttu-id="c2463-123">Modul runtime považuje názvy sestavení za nerozlišování velkých a malých písmen při vytváření vazby na sestavení, ale zachová bez ohledu na případ použití v názvu sestavení.</span><span class="sxs-lookup"><span data-stu-id="c2463-123">The runtime treats assembly names as case-insensitive when binding to an assembly, but preserves whatever case is used in an assembly name.</span></span> <span data-ttu-id="c2463-124">Několik nástrojů v Windows SDK zpracovává názvy sestavení při rozlišování velkých a malých písmen.</span><span class="sxs-lookup"><span data-stu-id="c2463-124">Several tools in the Windows SDK handle assembly names as case-sensitive.</span></span> <span data-ttu-id="c2463-125">Pro dosažení nejlepších výsledků spravujte názvy sestavení, jako by se jednalo o velká a malá písmena.</span><span class="sxs-lookup"><span data-stu-id="c2463-125">For best results, manage assembly names as though they were case-sensitive.</span></span>

## <a name="name-application-components"></a><span data-ttu-id="c2463-126">Pojmenovat součásti aplikace</span><span class="sxs-lookup"><span data-stu-id="c2463-126">Name application components</span></span>

 <span data-ttu-id="c2463-129">Například pokud máte sestavení s názvem *MyAssembly. exe* , které odkazuje na sestavení s názvem *MyAssembly. dll*, vazba proběhne správně, pokud spustíte *MyAssembly. exe*.</span><span class="sxs-lookup"><span data-stu-id="c2463-129">For example, if you have an assembly called *myAssembly.exe* that references an assembly called *myAssembly.dll*, binding occurs correctly if you execute *myAssembly.exe*.</span></span> <span data-ttu-id="c2463-130">Pokud však jiná aplikace provádí *MyAssembly. exe* pomocí metody <xref:System.AppDomain.ExecuteAssembly%2A?displayProperty=nameWithType> , modul runtime určí, že `myAssembly` je již načten, když aplikace *MyAssembly. exe* vyžádá vazbu `myAssembly` .</span><span class="sxs-lookup"><span data-stu-id="c2463-130">However, if another application executes *myAssembly.exe* using the method <xref:System.AppDomain.ExecuteAssembly%2A?displayProperty=nameWithType>, the runtime determines that `myAssembly` is already loaded when *myAssembly.exe* requests binding to `myAssembly`.</span></span> <span data-ttu-id="c2463-131">V tomto případě není *MyAssembly. dll* nikdy načtena.</span><span class="sxs-lookup"><span data-stu-id="c2463-131">In this case, *myAssembly.dll* is never loaded.</span></span> <span data-ttu-id="c2463-132">Protože *MyAssembly. exe* neobsahuje požadovaný typ, <xref:System.TypeLoadException> dojde k chybě.</span><span class="sxs-lookup"><span data-stu-id="c2463-132">Because *myAssembly.exe* does not contain the requested type, a <xref:System.TypeLoadException> occurs.</span></span>

 <span data-ttu-id="c2463-133">Chcete-li se tomuto problému vyhnout, ujistěte se, že sestavení, která tvoří vaši aplikaci, nemají stejný název sestavení nebo umísťují sestavení se stejným názvem do různých adresářů.</span><span class="sxs-lookup"><span data-stu-id="c2463-133">To avoid this problem, make sure the assemblies that make up your application do not have the same assembly name or place assemblies with the same name in different directories.</span></span>

> <span data-ttu-id="c2463-134">Pokud v .NET Framework umístíte sestavení se silným názvem do globální mezipaměti sestavení (GAC), název souboru sestavení musí odpovídat názvu sestavení, včetně přípony názvu souboru, například *. exe* nebo *. dll*.</span><span class="sxs-lookup"><span data-stu-id="c2463-134">In the .NET Framework, if you put a strong-named assembly in the global assembly cache, the assembly's file name must match the assembly name, not including the file name extension, such as *.exe* or *.dll*.</span></span> <span data-ttu-id="c2463-135">Například pokud název souboru sestavení je *MyAssembly. dll*, název sestavení musí být `myAssembly` .</span><span class="sxs-lookup"><span data-stu-id="c2463-135">For example, if the file name of an assembly is *myAssembly.dll*, the assembly name must be `myAssembly`.</span></span> <span data-ttu-id="c2463-136">Soukromá sestavení nasazená pouze v kořenovém adresáři aplikace mohou mít název sestavení, který se liší od názvu souboru.</span><span class="sxs-lookup"><span data-stu-id="c2463-136">Private assemblies deployed only in the root application directory can have an assembly name that is different from the file name.</span></span>
