### <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a><span data-ttu-id="ca86b-101">Metody SSE a SSE2 CompareGreaterThan správně zpracovávají vstupy NaN.</span><span class="sxs-lookup"><span data-stu-id="ca86b-101">SSE and SSE2 CompareGreaterThan methods properly handle NaN inputs</span></span>

<span data-ttu-id="ca86b-102">Následující <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> metody a <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> byly opraveny tak, aby správně `NaN` zpracovaly vstupy a odpovídaly hardwarovému chování ekvivalentních metod <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> ve třídě:</span><span class="sxs-lookup"><span data-stu-id="ca86b-102">The following <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> and <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> methods have been fixed to properly handle `NaN` inputs and match the hardware behavior of the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> class:</span></span>

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`
* `CompareScalarGreaterThanOrEqual`
* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

#### <a name="change-description"></a><span data-ttu-id="ca86b-103">Popis změny</span><span class="sxs-lookup"><span data-stu-id="ca86b-103">Change description</span></span>

<span data-ttu-id="ca86b-104">Dříve byly `NaN` vstupy na uvedené <xref:System.Runtime.Intrinsics.X86.Sse> a <xref:System.Runtime.Intrinsics.X86.Sse2> metody nesprávného výsledku.</span><span class="sxs-lookup"><span data-stu-id="ca86b-104">Previously, `NaN` inputs to the listed <xref:System.Runtime.Intrinsics.X86.Sse> and <xref:System.Runtime.Intrinsics.X86.Sse2> methods returned an incorrect result.</span></span> <span data-ttu-id="ca86b-105">Výsledek se také liší od výsledku generovaného odpovídající metodou ve <xref:System.Runtime.Intrinsics.X86.Avx> třídě.</span><span class="sxs-lookup"><span data-stu-id="ca86b-105">The result also differed from the result generated by the corresponding method in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="ca86b-106">Počínaje rozhraním .NET 5,0 tyto metody správně zpracovávají `NaN` vstupy a vracejí stejné výsledky jako odpovídající metody ve <xref:System.Runtime.Intrinsics.X86.Avx> třídě.</span><span class="sxs-lookup"><span data-stu-id="ca86b-106">Starting in .NET 5.0, these methods correctly handle `NaN` inputs and return the same results as the corresponding methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="ca86b-107">Standardní architektury prostředí Streaming SIMD Extensions (SSE) a Streaming SIMD Extensions 2 (SSE2) neposkytují přímou hardwarovou podporu pro tyto metody porovnání, takže se implementují v softwaru.</span><span class="sxs-lookup"><span data-stu-id="ca86b-107">The Streaming SIMD Extensions (SSE) and Streaming SIMD Extensions 2 (SSE2) industry standard architectures (ISAs) don't provide direct hardware support for these comparison methods, so they're implemented in software.</span></span> <span data-ttu-id="ca86b-108">Dříve byly metody nesprávně implementovány a nesprávně zpracovaly `NaN` vstupy.</span><span class="sxs-lookup"><span data-stu-id="ca86b-108">Previously, the methods were improperly implemented, and they incorrectly handled `NaN` inputs.</span></span> <span data-ttu-id="ca86b-109">Pro kód, který je nativně z nativního, může chybné chování způsobit chyby.</span><span class="sxs-lookup"><span data-stu-id="ca86b-109">For code ported from native, the incorrect behavior may introduce bugs.</span></span> <span data-ttu-id="ca86b-110">Pro 256 cestu kódu mohou metody také vytvořit různé výsledky pro ekvivalentní metody ve <xref:System.Runtime.Intrinsics.X86.Avx> třídě.</span><span class="sxs-lookup"><span data-stu-id="ca86b-110">For a 256-bit code path, the methods can also produce different results to the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="ca86b-111">Jako příklad, jak byly metody dříve nesprávné, můžete implementovat `CompareNotGreaterThan(x,y)` jako `CompareLessThanOrEqual(x,y)` pro běžná celá čísla.</span><span class="sxs-lookup"><span data-stu-id="ca86b-111">As an example of how the methods were previously incorrect, you can implement `CompareNotGreaterThan(x,y)` as `CompareLessThanOrEqual(x,y)` for regular integers.</span></span> <span data-ttu-id="ca86b-112">U `NaN` vstupů však tato logika počítá špatný výsledek.</span><span class="sxs-lookup"><span data-stu-id="ca86b-112">However, for `NaN` inputs, that logic computes the wrong result.</span></span> <span data-ttu-id="ca86b-113">Místo toho použití `CompareNotLessThan(y,x)` porovná čísla správně *a* bere `NaN` v potaz vstupy.</span><span class="sxs-lookup"><span data-stu-id="ca86b-113">Instead, using `CompareNotLessThan(y,x)` compares the numbers correctly *and* takes `NaN` inputs into consideration.</span></span>
